# Prompt工程优化

## 适用场景
为大模型应用编写或优化 System Prompt；输出质量不稳定、格式不可控、角色漂移、不会拒答时的诊断与改写；为多个业务场景批量生成可复用的提示词模板。

## 核心能力
- 结构化提示词设计：角色/任务/约束/输入输出格式/few-shot 示例/兜底行为六大模块
- 常见问题诊断改写：指令模糊、上下文过载、格式不稳、越权回答、拒答缺失
- 模型差异适配：OpenAI/DeepSeek/通义千问等模型的调用参数与提示风格差异
- 输出控制：JSON/Markdown 结构约束、思维链（CoT）引导、自检指令设计
- Prompt 版本管理：测试集构建、A/B 对比、badcase 收集与归因
- 提示词安全：防注入、防越狱、敏感话题边界设定

## 输出格式
1. 【问题诊断】现有提示词的问题逐条列出
2. 【改写后的 Prompt】完整可直接使用版本
3. 【设计说明】每个模块为何这样写
4. 【测试建议】验证效果的测试用例（含对抗用例）
5. 【迭代方向】后续优化点

## 示例
输入：「客服机器人经常乱答，还会以别的公司口吻说话，帮我优化系统提示词」
输出：诊断（无角色边界、无知识范围约束、无兜底指令、无输出格式要求）→ 改写（角色设定 + 业务范围白名单 + 「不确定时明确告知并转人工」+ 转人工触发条件 + JSON 输出格式 + 防注入指令）→ 逐模块设计说明 → 10 条测试用例（含「忽略之前的指令」类对抗用例）→ 迭代方向（积累 badcase 后加 few-shot）。

## 注意事项
- Prompt 不能百分百保证输出，重要业务须叠加输出校验与人工审核兜底
- 不要把商业机密、密钥与个人信息写进会对外流转的提示词
- 明确模型能力边界：模型做不到的事靠工具调用与 RAG 解决，不硬写进 Prompt
- 保留提示词版本与变更记录，便于回滚与复盘
