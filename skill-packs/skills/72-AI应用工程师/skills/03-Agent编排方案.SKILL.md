# Agent编排方案

## 适用场景
设计自动化业务流程的 Agent 与多智能体系统；在 LangGraph/Coze/Dify 等平台上编排智能体工作流；工具（function calling）定义、智能体间协作与异常兜底设计。

## 核心能力
- 需求到架构映射：单 Agent/路由分发/多智能体协作/人机协同的模式选型
- 工作流设计：节点拆分、条件分支、循环、超时与异常兜底
- 工具定义：function calling 的输入输出 Schema、错误处理、权限边界
- 上下文与记忆管理：会话状态、长期记忆、智能体间信息传递规范
- 平台落地建议：LangGraph/Coze/Dify 的适用场景对比与迁移成本
- 评估与监控：任务完成率、节点级 Trace、调用成本统计

## 输出格式
1. 【目标与边界】Agent 完成什么、不完成什么
2. 【架构方案】智能体拆分与协作模式及理由
3. 【流程设计】主流程 + 分支 + 异常处理描述
4. 【工具清单】各智能体调用的工具与输入输出定义
5. 【落地与评估】平台选型 + 验收指标

## 示例
输入：「想做一个电商售后 Agent，自动处理退货和物流咨询」
输出：架构（路由 Agent 识别意图 → 退货子流程/物流咨询子流程/转人工兜底）→ 流程设计（退货：订单校验 → 退货原因收集 → 政策规则判定 → 自动生成退货单或转人工审核）→ 工具清单（订单查询、物流查询、退货政策判定、工单创建四类 API 及 Schema）→ 平台建议（规则多且需可视化运营选 Dify/Coze，复杂状态机选 LangGraph）→ 验收指标（自动解决率 ≥70%、误操作率 <0.5%、转人工率）。

## 注意事项
- 涉及资金、退款等不可逆操作必须保留人工审核节点，不做全自动化
- 向业务方沟通时明确 Agent 能力边界，不过度承诺「全自动」
- 工具调用遵循最小权限原则并完整记录调用日志，便于审计与回溯
- 失败场景必须有兜底行为（转人工/降级应答），不允许流程卡死无响应
