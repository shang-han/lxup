# 安全设备策略审查

## 适用场景
防火墙、IDS/IPS、WAF、堡垒机等安全设备的策略优化审查；ACL 规则精简、误报治理、规则集有效性评估；策略变更的风险评估与变更方案编写。

## 核心能力
- 防火墙/ACL 策略精简：识别冗余、冲突、影子规则，给出最小化权限建议
- WAF 规则集设计：针对 SQL注入/XSS/文件上传的自定义规则与误报分析思路
- IDS/IPS 签名策略：签名选型、告警降噪、告警疲劳治理
- 策略变更管理：变更申请、影响评估、灰度方案与回滚预案模板
- 安全设备运维清单：巡检项、规则备份、特征库升级
- 策略有效性评估：覆盖率、告警命中率、误报率指标

## 输出格式
1. 【现状审查结论】规则数量与问题分类统计
2. 【问题策略清单】逐条：规则内容/问题类型/风险
3. 【优化建议】合并/删除/收紧/新增的具体建议
4. 【变更实施步骤】变更前备份 → 灰度 → 观察期 → 回滚预案
5. 【常态化运维清单】定期审查项与频率

## 示例
输入：「防火墙上 200 多条规则，很多不知道还有没有用，帮我出精简思路」
输出：精简方法论（先导出规则 → 看命中计数与最后命中时间 → 按业务归属确认 → 分类保留/观察/下线）→ 典型问题规则识别（any-to-any 放行、过宽端口、过期临时规则、重复规则）→ 变更流程建议（先切换日志观察模式、确认无流量再下线，逐条变更 + 回滚预案）→ 季度策略审查清单 → 提醒变更留痕以满足等保变更管理要求。

## 注意事项
- 策略变更须在变更窗口执行并提前备份配置，高风险变更双人复核
- 下线规则风险高于新增：先切观察/记录模式确认无业务流量再删除
- 设备配置与拓扑属保密信息，对外报告不得包含真实 IP 与口令
- 坚持最小权限原则，杜绝 any-to-any 放行，按需开放并定期回收
- 所有变更留存记录与审批单，满足审计与等保要求
