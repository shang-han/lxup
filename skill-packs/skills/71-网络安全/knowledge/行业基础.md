# 行业基础

## 行业概况
网络安全是守护企业业务连续性与数据资产的岗位，覆盖攻防运维、合规建设、应急响应、安全开发等方向。受《网络安全法》《数据安全法》《个人信息保护法》与等保 2.0 驱动，安全已从「可选项」变为企业法定义务。攻击侧产业化（勒索即服务、钓鱼产业链），防守侧从合规驱动转向实战化对抗（HW 攻防演练常态化），渗透测试、安全运营、合规咨询人才缺口持续存在。

## 专业技能体系
### 硬技能
- 渗透测试与漏洞扫描（SQL注入/XSS/CSRF/文件上传）
- 安全设备运维（防火墙/IDS/WAF）
- 日志分析与安全事件溯源
- 等保2.0合规要求
- 应急响应流程
### 软技能
- 安全风险翻译成业务语言
- 安全意识培训

## 常用术语
- 等保2.0：网络安全等级保护制度（GB/T 22239），系统定级后须备案并通过测评
- CVSS：通用漏洞评分系统，0-10 分衡量漏洞严重度
- 0day：厂商尚未发现或未修复的漏洞，危害极大
- WAF：Web 应用防火墙，拦截 SQL 注入/XSS 等 Web 攻击
- SIEM/SOC：安全信息与事件管理平台/安全运营中心，集中日志分析与告警处置
- ATT&CK：攻击者战术与技术知识库框架，用于防守覆盖度评估与事件溯源
- 红蓝对抗：模拟攻击方（红队）与防守方（蓝队）实战演练，检验真实安全水位
- 应急响应（IR）：安全事件发生后的抑制、排查、根除、恢复、复盘标准化流程

## 核心工作流程
日常运营：安全设备巡检 → 告警处置与调优 → 漏洞跟踪闭环 → 安全需求响应
周期性：漏洞扫描/渗透测试 → 整改复测 → 策略审查优化 → 意识培训与钓鱼演练
事件驱动：监测告警 → 抑制隔离 → 排查溯源 → 根除恢复 → 复盘报告 → 加固
合规线：系统定级备案 → 差距分析 → 整改 → 等保测评 → 监督检查

## 常见问题
Q：漏洞很多修不过来，怎么排优先级？
A：两个维度相乘：CVSS 等级 × 业务暴露面。优先修互联网暴露面的高危且有在野利用的漏洞，内网高危次之；开发修复前先用 WAF 虚拟补丁争取时间。

Q：中了勒索病毒第一时间做什么？
A：先断网隔离（拔网线/隔离主机）防止扩散，再保护现场（保留日志与加密样本用于溯源和解密查询），然后排查根除，最后恢复业务。切忌直接重启或格式化，会毁掉溯源证据。

Q：怎么向不懂技术的领导说明安全风险？
A：把技术语言翻译成业务语言：不说「存在 SQL 注入」，而说「这个漏洞被利用会导致客户数据泄露，按数据安全法最高可罚上年营收 5%，还要通知全部受影响用户」，用损失量化代替技术堆砌。

Q：渗透测试的法律边界是什么？
A：授权是前提（书面授权明确范围与时间窗），验证须无害（不投放破坏性 payload），数据不触碰（不拖取业务数据），漏洞先报客户不对外披露。未授权的渗透无论是否造成损失均属违法。
