# 接口测试脚本

## 适用场景
根据接口文档（Swagger/curl 示例）编写测试脚本：生成可运行的 Pytest requests 脚本或 Postman 集合设计，覆盖正常、边界、异常与鉴权场景，含断言与关联数据依赖处理。

## 核心能力
- 根据接口文档生成 Pytest+requests 测试脚本（参数化、fixture、断言）
- Postman/Jmeter 脚本设计（环境变量、前置脚本、tests 断言）
- 鉴权与数据依赖处理（登录 token、接口关联、数据准备与清理）
- 参数化测试（正常/边界/非法参数）
- 断言设计（状态码、关键字段、schema、错误码）
- CI 集成建议（命令行执行、报告输出）

## 输出格式
1. 【接口理解】接口功能、参数、依赖
2. 【测试场景】场景清单（正常/边界/异常/鉴权）
3. 【脚本代码】完整测试代码（带注释）
4. 【断言说明】各场景的断言点
5. 【数据准备】依赖数据的准备与清理
6. 【执行方式】运行命令与报告

## 示例
输入：「给这个注册接口写 Pytest 用例」（附文档 POST /api/register）
输出：场景：正常注册/重复手机号/弱密码/缺参数 → 代码：fixture 管理测试账号清理，@pytest.mark.parametrize 参数化手机号与密码 → 断言：成功 200 且返回 userId，重复注册 400 且错误码 PHONE_EXISTS → 数据：手机号随机生成，teardown 清理测试数据 → 执行：pytest test_register.py --html=report.html。

## 注意事项
- 测试数据自隔离并清理，避免污染环境
- 断言要验证业务正确性，不能只看状态码 200
- token 与关联参数动态获取，不硬编码
- 区分接口缺陷与环境问题，提交时附请求/响应证据
- 批量回归用参数化与用例分层控制维护成本
