# 测试用例设计

## 适用场景
根据需求文档/接口文档/页面描述设计测试用例：选择恰当的用例设计方法，覆盖正常、边界、异常与组合场景，输出可执行的用例清单。

## 核心能力
- 等价类划分（有效/无效等价类）与边界值分析
- 场景法（基本流 + 备选流 + 异常流）
- 正交表法简化多因素组合
- 错误推测（常见坑：空值、特殊字符、并发、超时、权限）
- 用例优先级标注（P0 冒烟/P1 核心/P2 一般）
- 标准格式输出（编号/前置条件/步骤/预期/优先级）

## 输出格式
1. 【测试点分析】功能点与风险点拆解
2. 【用例清单】编号、前置条件、步骤、预期结果、优先级、设计方法
3. 【边界与异常】边界值与异常场景的覆盖说明
4. 【组合场景】正交组合或场景流用例
5. 【冒烟集】P0 冒烟用例子集
6. 【遗漏提示】需求不明确处的补充建议

## 示例
输入：「给『手机号+密码登录』设计用例」
输出：测试点：账号校验、密码校验、错误提示、安全性 → 清单：有效等价类（正常登录 P0）→ 边界：手机号 10/11/12 位、密码 5/6/20/21 位 → 异常：密码错误、未注册、连续 5 次锁定、特殊字符注入 → 场景：忘记密码→重置→登录全链路 → 冒烟集 8 条 → 待确认：是否支持国际区号。

## 注意事项
- 用例先覆盖正常主流程，再覆盖边界与异常
- 一条用例验证一个点，预期结果明确可判断
- 无效等价类也要有对应的提示预期
- 多因素组合用正交表降维，避免组合爆炸
- 结合历史缺陷数据补充错误推测用例
