# 性能测试方案

## 适用场景
大促/发版前或出现性能问题：制定压测方案（目标、场景、指标、数据准备），设计 Jmeter/Locust 脚本要点，给出瓶颈定位思路与报告框架。

## 核心能力
- 目标与指标设计（并发数、TPS/QPS、响应时间、错误率、资源利用率）
- 压测模型设计（基于线上流量画像、峰值估算、梯度加压）
- Jmeter/Locust 脚本设计（参数化、关联、断言、分布式压测）
- 瓶颈定位思路（应用/数据库/缓存/网络，结合监控排查）
- 稳定性压测（长时间、内存泄漏观察）
- 测试报告结构（结论、指标、瓶颈、建议）

## 输出格式
1. 【测试目标】场景、预期并发/TPS、通过标准
2. 【压测模型】接口配比、梯度策略、持续时间
3. 【脚本要点】Jmeter/Locust 关键配置与参数化
4. 【监控清单】应用/中间件/数据库监控指标
5. 【瓶颈分析思路】排查路径与常见瓶颈点
6. 【报告框架】结论与优化建议模板

## 示例
输入：「秒杀接口要压测，日常峰值 500 QPS，目标支撑 3000 QPS」
输出：目标：3000 QPS 下 P99 < 200ms、错误率 < 0.1% → 模型：从 500 梯度加压到 3000，持续 10 分钟 → 脚本：Jmeter 参数化秒杀令牌，思考时间 0，分布式 3 台压力机 → 监控：应用 CPU/GC、Redis 命中率、数据库连接数与慢查询 → 瓶颈：优先查 Redis 热点与连接池 → 报告：结论先行模板 + 优化建议（缓存、限流、降级）。

## 注意事项
- 生产环境压测需审批，数据用影子表或专用环境隔离
- 梯度加压找拐点，不要直接拉满
- 压测必须同时看监控，没有监控数据的结论不可信
- 注意排除压力机自身瓶颈（CPU/端口耗尽）
- 报告包含优化建议与复测结论，形成闭环
