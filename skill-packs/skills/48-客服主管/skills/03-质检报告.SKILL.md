# 质检报告

## 适用场景
客服主管需要定期对客服对话进行质量抽检，输出结构化的质检报告，评估服务质量、发现问题、为培训与考核提供依据。

## 核心能力
- 质检维度设计：服务态度、响应时效、专业度、规范用语、问题解决率
- 评分标准：量化评分表与扣分项定义
- 抽检方法：随机抽检 + 重点抽检（投诉单、长会话）
- 问题分类：态度类、专业类、流程类、违规类
- 团队与个人对比：识别共性问题与个体短板
- 改进建议：针对性培训与辅导方向

## 输出格式
1. 【质检概览】抽检数量、周期、整体得分
2. 【评分维度】各维度得分与权重
3. 【问题分类】高频问题统计与典型案例
4. 【团队表现】整体水平与排名
5. 【个人表现】优秀与待改进名单
6. 【改进建议】培训与流程优化方向

## 示例
输入：「帮我做一份本月客服质检报告，抽检了 200 条会话」
输出：概览「本月抽检 200 条，综合得分 87 分（环比 +3）」→ 维度（态度 92、时效 85、专业 84、规范 88）→ 高频问题（① 产品参数答错占 30% ② 高峰期响应超时占 25%）→ 团队（整体良好，专业度是短板）→ 个人（A 组 90 分领先，客服 X 专业分偏低需辅导）→ 建议（开展产品知识专项培训、高峰时段增配人手、建立参数速查表）。

## 注意事项
- 评分标准要事先公示，保证公平可比
- 抽检样本要有代表性，兼顾随机与重点
- 质检目的是改进而非惩罚，反馈要对事不对人
- 典型案例脱敏处理，保护客户隐私
- 质检结果与培训、考核形成闭环
