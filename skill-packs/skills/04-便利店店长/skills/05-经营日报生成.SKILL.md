# 经营日报生成

## 适用场景
便利店/超市店长每日营业结束后，需要把当天的销售额、来客数、客单价、品类表现、损耗等数据整理成规范日报，便于掌握经营状况、发现问题并上报。

## 核心能力
- 汇总销售数据：总营业额、来客数、客单价
- 分渠道/支付：现金、移动支付、会员消费占比
- 品类表现：各品类销售与毛利贡献
- 损耗与报废：当日损耗、临期报废金额
- 同环比对比：对比昨日/上周同期，标异常
- 问题与待办：异常归因与次日改进项

## 输出格式
1. 【基本信息】门店 / 日期 / 填报人
2. 【销售概览】营业额、来客数、客单价（含同环比）
3. 【结构分析】支付方式 / 会员 / 品类占比
4. 【损耗报废】当日损耗金额与原因
5. 【异常与问题】标红项 + 归因
6. 【次日待办】改进动作清单

## 示例
输入：「今天营业额 6800，来客 520，会员消费占 40%，报废了 150 元的鲜食」
输出：营业额 6800（环比 +5%）→ 客单价约 13 元 → 来客 520 → 会员占比 40%（建议继续提升会员转化）→ 损耗：鲜食报废 150 元（备货偏多）→ 异常：鲜食损耗偏高 → 待办：明日下调鲜食订货量、临期提前打折，跟踪损耗变化。

## 注意事项
- 日结数据真实准确，账实相符，差异及时核查
- 同环比口径一致，避免因节假日误判趋势
- 损耗要记清原因，作为订货调整依据
- 异常项落到具体改进动作，而非笼统描述
