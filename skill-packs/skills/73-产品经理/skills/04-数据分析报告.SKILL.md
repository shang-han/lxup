# 数据分析报告

## 适用场景
版本效果复盘、转化漏斗分析、留存与活跃分析、指标异常排查；埋点方案与指标体系设计；给管理层的数据报告撰写。

## 核心能力
- 指标体系设计：北极星指标、OSM 模型（目标-策略-度量）、指标树拆解
- 埋点方案：事件定义、参数设计、验证方法、埋点文档模板
- 漏斗分析：步骤转化率、瓶颈定位、分群与版本对比
- 留存分析：留存曲线、同期群（Cohort）分析、关键行为与留存的相关性
- 异常排查思路：指标拆解 → 维度下钻 → 变量定位 → 假设验证
- 报告撰写：结论先行、数据配解读、行动建议可执行

## 输出格式
1. 【分析背景与问题】要回答的业务问题
2. 【分析思路】指标拆解与下钻维度
3. 【数据发现】关键数据与解读，结论先行
4. 【原因归因】可能原因、证据与置信度
5. 【行动建议】按优先级排序的改进项

## 示例
输入：「注册转化率上周从 40% 掉到 25%，帮我排查」
输出：思路（按渠道/设备/版本/时间段逐层下钻）→ 假设清单（渠道结构变化？新版本流程改动？验证码到达率？竞品活动？）→ 每个假设的验证方法（对比各渠道转化、看版本发布时间点、查短信发送日志）→ 结论模板（结论 + 数据证据 + 置信度）→ 行动建议（若定位为新版本流程问题，建议回滚或开 A/B 验证后再全量）。

## 注意事项
- 相关性不等于因果：归因前排除混杂因素（渠道结构、节假日、大盘波动）
- 绝对值与比率一起看，避免小样本波动误导决策
- 埋点须前置规划，需求评审时同步埋点方案，别上线后补数
- 数据分析须符合《个人信息保护法》，不使用超出授权范围的数据
