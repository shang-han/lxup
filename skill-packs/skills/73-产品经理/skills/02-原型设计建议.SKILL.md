# 原型设计建议

## 适用场景
动手画原型前快速产出页面结构与交互流程建议；对现有原型或设计稿做可用性评审；新功能的信息架构与导航设计；Axure/Figma 落地思路与组件复用建议。

## 核心能力
- 信息架构设计：功能组织、导航层级、页面树
- 交互流程设计：任务流、状态流转、页面跳转关系与回退逻辑
- 页面布局建议：信息优先级、模块排布、常见组件复用
- 常见交互模式库：列表/搜索/筛选/表单/反馈的适用场景
- 原型评审：基于尼尔森可用性十原则逐项检查
- 状态完整性：空状态、加载、失败、无权限页面的设计提醒

## 输出格式
1. 【目标与用户路径】核心任务流程描述
2. 【信息架构】页面树与导航结构
3. 【页面结构建议】主要页面的模块与布局理由
4. 【交互细节】关键状态、反馈方式、异常处理
5. 【评审与优化点】针对现有方案的问题清单

## 示例
输入：「电商小程序的退货流程，原型怎么设计？」
输出：用户路径（订单详情 → 申请退货 → 选择原因与凭证 → 选择退货方式 → 进度跟踪）→ 每页结构与关键字段建议（原因预设选项 + 自定义、凭证拍照上传、运费承担方提示）→ 交互细节（申请成功后状态变更推送、物流单号自动回填、超时自动同意规则提示）→ 常见坑（退货入口层级过深、进度状态反馈不及时、部分退货规则未说明）→ 评审清单。

## 注意事项
- 原型服务于沟通对齐：核心流程先用低保真对齐，别过早纠结视觉细节
- 从一开始就考虑异常态：空数据、加载失败、无权限的页面不能缺
- 移动端注意手势操作与 PC 端点击习惯的差异
- 优先复用现有组件库，降低开发与测试成本
