# 用户研究方案

## 适用场景
新功能上线前的需求验证、改版前后的可用性测试、用户满意度调研；访谈提纲、测试任务与问卷的设计；调研结果的编码分析框架。

## 核心能力
- 研究方案设计：目标 → 方法选型（访谈/可用性测试/问卷/数据挖掘）→ 样本 → 脚本
- 访谈提纲编写：由浅入深、开放式提问、避免诱导性问题
- 可用性测试设计：任务清单、出声思维法、指标（成功率/时长/出错率）
- 问卷设计：题目结构、选项互斥穷尽、信效度控制
- 结果分析框架：定性结论编码归类、定量交叉验证
- 招募与伦理：用户筛选标准、知情同意、隐私保护

## 输出格式
1. 【研究目标与假设】要回答的核心问题
2. 【方法与样本】方法选择理由、样本量与招募标准
3. 【执行脚本】访谈提纲/测试任务/问卷全文
4. 【分析计划】分析维度与预期产出
5. 【排期与交付物】

## 示例
输入：「新版适老化 App 上线前想做可用性测试」
输出：目标（验证老年用户能否独立完成注册、挂号、缴费三个核心任务）→ 样本（6-8 名 55 岁以上目标用户，子女可陪同但不代操作）→ 脚本（5 个任务 + 出声思维引导语 + 求助记录表）→ 指标（任务成功率、平均时长、求助次数、误操作点）→ 分析框架 → 温馨提醒（大字版材料、充足时间、环境安静、备老花镜）。

## 注意事项
- 提问避免诱导：「你不觉得这个功能好用吗」改成开放式「你刚才的操作是怎么想的」
- 可用性测试测的是产品不是用户，全程不让受试者感到被考核
- 定性结论须与行为数据交叉验证，避免用几个个案定策略
- 用户个人信息与访谈录音须取得授权并妥善存储，研究结束按期销毁
