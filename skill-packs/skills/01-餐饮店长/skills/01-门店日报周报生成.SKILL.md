# 门店日报周报生成

## 适用场景
餐饮店长每日打烊后、每周复盘时，需要把零散的营收、客流、成本数据整理成一份可直接上报老板或区域经理的规范报表，并附上问题分析和改进动作。

## 核心能力
- 汇总营收结构：堂食 / 外卖 / 团购核销分渠道统计
- 计算核心指标：营业额、客单价、来客数、翻台率、毛利率
- 成本与损耗：食材成本率、人工成本率、能耗、当日损耗金额
- 同环比分析：对比昨日 / 上周同期，标出异常波动项
- 问题归因：客流下滑、某菜品退单多、外卖差评集中等定位
- 生成改进清单：可执行、可追踪的次日 / 下周动作

## 输出格式
1. 【基本信息】门店 / 日期 / 填报人
2. 【经营概览】营业额、来客数、客单价、翻台率（含同环比）
3. 【渠道结构】堂食 / 外卖 / 团购占比
4. 【成本分析】食材 / 人工 / 能耗成本率、当日损耗
5. 【异常与问题】标红项 + 原因初判
6. 【改进动作】明日/下周待办清单（责任人 + 时限）

## 示例
输入：「今天营业额 12800，堂食 8600 外卖 4200，来客 320 人，退单 3 笔都是酸菜鱼太咸」
输出：营业额 12800（环比 -6%）→ 客单价 40 元 → 堂食外卖占比 67%:33% → 异常：酸菜鱼退单 3 笔（口味偏咸）→ 改进动作：明早检查酸菜鱼腌料配比，由后厨组长复核，连续 3 天跟踪退单数。

## 注意事项
- 数据须真实，不得为好看而调整或估算关键指标
- 同环比口径保持一致（同星期对比而非简单相邻日）
- 异常项要落到「原因 + 动作 + 责任人」，避免只写「需加强」
- 涉及员工绩效的数据注意保密，仅限管理层查看
