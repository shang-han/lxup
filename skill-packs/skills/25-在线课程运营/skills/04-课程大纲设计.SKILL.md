# 课程大纲设计

## 适用场景
课程运营/教研在开发新课或优化旧课时，需要设计结构清晰、循序渐进、颗粒度合理的课程大纲，兼顾教学效果、完课率与卖点呈现。

## 核心能力
- 学习目标设计：用可衡量的结果描述（学完能做什么）
- 章节结构搭建：由浅入深、模块化、先易后难的递进逻辑
- 课时颗粒度控制：单课时长建议（录播 10-15 分钟为宜）与拆分
- 内容完整性自查：从入门到应用的闭环，避免断层
- 练习与作业嵌入：每章配套实践，强化吸收
- 卖点与大纲结合：大纲既是教学框架也是销售素材
- 不同品类适配：技能实操类 vs 知识体系类 vs 训练营的结构差异

## 输出格式
1. 【课程定位】目标人群 + 学习目标
2. 【整体结构】模块划分与递进逻辑
3. 【详细大纲】章节 + 课时 + 内容要点
4. 【练习设计】各章配套作业/实操
5. 【课时规划】总课时与单课时长
6. 【亮点说明】可用于宣传的课程特色

## 示例
输入：「我想做一门面向零基础的小白Python入门课，帮我设计大纲」
输出：定位（零基础、能独立写小工具）→ 学习目标（学完能写自动化脚本、爬取简单数据）→ 结构（环境搭建→基础语法→数据结构→函数与模块→实战项目）→ 详细大纲（5 模块、约 30 节、每节 12-15 分钟）→ 练习（每章配小练习、每模块一个实战小项目）→ 课时规划 → 亮点（项目驱动、配套代码、答疑社群）。

## 注意事项
- 以学习者视角设计，循序渐进，避免一上来难度过高劝退
- 单课时长控制，过长影响完课率，碎片化更易坚持
- 每章有明确小目标与练习，形成「学习—实践—反馈」闭环
- 大纲要能支撑宣传卖点，避免货不对板
- 预留迭代空间，根据学员数据反馈持续优化
