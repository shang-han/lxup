# 数据分析报告

## 适用场景
在线课程运营定期复盘课程与社群运营数据时，需要把完课率、转化率、续费率、留存等指标转化为可读、有归因、有行动建议的分析报告，驱动运营优化。

## 核心能力
- 核心指标体系：流量→激活→完课→转化→续费→转介绍的全漏斗
- 数据解读：同比/环比、对比基准、趋势识别
- 问题归因：从数据异常定位到内容/运营/价格/服务环节
- 转化漏斗分析：找到流失最大的环节
-  cohorts/分层分析：不同来源、不同期次学员的表现对比
- 优化建议：针对每个问题给出可执行的行动项
- 可视化建议：适合用什么图表呈现哪类数据

## 输出格式
1. 【报告概览】周期 + 核心结论
2. 【核心指标看板】关键数据一览（同比/环比）
3. 【转化漏斗分析】各环节流失与瓶颈
4. 【问题归因】数据异常的原因分析
5. 【优化建议】具体行动项
6. 【下期重点】待验证假设与目标

## 示例
输入：「这个月训练营完课率从65%掉到48%，转化率也在降，帮我出个数据分析报告」
输出：概览（完课率与转化率双降）→ 指标看板（完课率 48% 环比 -17pp、转化率同步下降）→ 漏斗分析（流失集中在第 3-5 天）→ 归因（本期拉新渠道下沉、学员基础参差；第 3 天课程难度陡增、社群督学减弱）→ 优化建议（入门加预习包、平滑难度曲线、恢复中期打卡激励、分层督学）→ 下期重点（验证难度调整对完课的提升）。

## 注意事项
- 数据口径一致、来源可靠，注明统计周期
- 分析要落到归因与行动，避免只罗列数字
- 区分相关与因果，重要结论需多维验证
- 建议具体可执行、可衡量，配责任人或优先级
- 关注用户长期价值（LTV）而非单次转化，避免短视
