# 客户档案管理

## 适用场景
为新客建立皮肤与消费档案，记录每次护理情况与卡项余额，制定护理计划与回访节奏，唤醒沉睡客户，提升复购与到店率。

## 核心能力
- 建档模板：基本信息 / 皮肤类型与问题 / 过敏史 / 护理目标 / 会员卡类型与余额 / 客户分级（A/B/C 客）
- 护理记录结构化：日期、项目、美容师、皮肤反应、效果对比、家居建议、下次计划
- 疗程计划表：疗程目标 / 建议频次 / 阶段节点 / 预期效果描述
- 回访提醒生成：预约前提醒、护理后 24-48 小时回访、卡余量不足提醒、久未到店唤醒，各配话术
- 客户标签与跟进策略：新客 / 活跃 / 沉睡 / 高价值，分别给跟进频率与动作建议

## 输出格式
- 档案类：结构化表单（键值清单或 Markdown 表格）
- 计划类：时间线表格（节点 / 目的 / 话术 / 记录栏）
- 提醒类：可直接复制使用的回访话术，口语化

## 示例
输入：「给王女士建档，干敏肌，刚买了 3980 的补水修护 10 次卡」
输出：档案表单（含过敏史栏、皮肤问题记录栏、卡余 10 次）→ 疗程计划（前 3 次每周 1 次打底、中段两周 1 次、后期月度维护）→ 回访话术：「王姐，做完当天别用功效型面膜，温水洗脸就行，明早起来皮肤可能会有点紧，是正常的补水反应……」→ 第 8 次时触发续卡提醒节点。

## 注意事项
- 顾客皮肤照片、联系方式、消费记录属个人信息，建档须告知用途并征得同意，资料不外传、不发群
- 档案只做记录与提醒，不对皮肤问题做医学诊断，疑似皮肤病建议顾客就医
- 信息采集最小化，不收集与服务无关的隐私
- 回访控制频率，同一顾客一周内主动联系不超过 2 次，避免打扰
