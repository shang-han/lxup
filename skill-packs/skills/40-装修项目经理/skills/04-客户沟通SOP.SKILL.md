# 客户沟通SOP

## 适用场景
项目经理/工长在签约、开工、施工、验收、售后各节点与业主沟通，尤其是增项、延误、质量异议、催款等敏感场景，需要成熟话术与标准动作来降低冲突、留存证据。

## 核心能力
- 全流程沟通节点标准化：签约（讲清范围与增项规则）→ 开工（建群、介绍工序与对接人）→ 施工（每周进度播报）→ 验收（约时间、讲标准）→ 增项（先报价后施工）→ 竣工（验收交底）→ 售后（质保响应）
- 敏感场景话术库：增项谈判、工期延误解释、质量异议回应、进度款催收
- 进度播报模板：微信群周报（现场照片 + 本周完成 + 下周计划 + 需业主配合事项）
- 投诉处理四步：接收记录 → 核实事实 → 给出方案与时限 → 跟踪闭环回访
- 关键承诺一律书面确认（微信文字/变更单），重要沟通留痕
- 提供禁忌话术对照表：哪些说法会埋纠纷雷

## 输出格式
1. 【场景识别】沟通所处阶段与类型
2. 【沟通目标】本次要达成的结果
3. 【话术正文】开场 → 关键说明 → 异议应对 → 收尾确认
4. 【书面确认事项】需微信/变更单落实的内容
5. 【禁忌提醒】容易引发纠纷的说法

## 示例
输入：「施工中发现原墙基层要铲除重做，客户觉得不该加钱，怎么沟通？」
输出：场景识别（增项谈判-质量必需项）→ 目标（业主理解铲除必要性并书面确认费用）→ 话术（先发现场照片说明原基层起砂空鼓 → 讲不铲除会导致后期开裂脱落的后果 → 出示报价明细与计价依据 → 引用合同增项条款 → 给选项：铲除重做 / 局部处理及各自风险）→ 书面确认（变更单签字或微信文字确认截图存档）→ 禁忌（不说「先干着，钱好商量」「这个本来就该加钱」）。

## 注意事项
- 增项必须施工前书面告知价格并取得客户确认，先斩后奏是装修纠纷第一来源
- 口头承诺（免费赠送、延保、口头减价）一律落文字，否则无效且伤信任
- 延误与质量问题主动先报、带上方案，不要等客户发现追问
- 沟通对事不对人，保持专业平和；敏感谈话保留聊天记录
- 款项沟通以合同节点为依据，催款提前提醒、就事论事不情绪化
