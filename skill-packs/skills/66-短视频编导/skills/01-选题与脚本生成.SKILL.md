# 选题与脚本生成

## 适用场景
账号缺乏选题灵感，或需要把客户产品卖点转化为短视频内容：生成选题创意库和完整拍摄脚本（钩子开头、口播/剧情、反转、结尾互动引导）。

## 核心能力
- 根据账号定位与目标人群生成选题创意库（痛点型/干货型/剧情型/热点型/对比测评型）
- 开头 3 秒钩子设计（悬念、冲突、结果前置、共鸣式提问）
- 脚本结构把控：钩子 → 铺垫 → 反转/干货 → 结尾引导，逐秒控制节奏与时长
- 把产品卖点翻译成用户语言和场景化台词
- 输出逐句脚本（时间码/画面/口播/字幕）
- 提供开头钩子的 2-3 个备选版本供 AB 测试

## 输出格式
1. 【选题清单】5-10 个选题创意，标注内容类型与目标人群
2. 【推荐选题】推荐理由与预期数据表现
3. 【脚本正文】逐条：时间码 / 画面 / 口播 / 字幕
4. 【钩子设计】开头 3 秒钩子及备选版本
5. 【反转/共鸣点】视频中的高潮设计
6. 【结尾引导】点赞、评论、关注引导话术

## 示例
输入：「帮便携榨汁杯写个脚本，主打上班族」
输出：选题清单（办公室下午茶场景、早餐省时对比、健身备餐干货…）→ 推荐选题「办公室 3 分钟早餐挑战」→ 逐句脚本：0-3s 钩子「同事以为我在炫富，其实这东西才 59」→ 3-15s 使用过程 → 15-25s 效果展示+反转 → 结尾引导评论「你们早餐吃什么」→ 附 3 个钩子备选版本。

## 注意事项
- 开头 3 秒必须进入钩子，避免冗长自我介绍和品牌铺垫
- 口播口语化，避免书面化表达和参数堆砌
- 一个脚本只打一个卖点，不要贪多
- 文案规避广告法禁用词（最、第一、百分百有效等）
- 推荐选题要给出预期数据表现的依据，不拍脑袋
