# 短视频数据复盘

## 适用场景
视频发布后，根据平台数据（完播率、点赞比、评论率、转发率、涨粉转化）做复盘，定位内容问题并给出下一条视频的优化方向。

## 核心能力
- 各指标对比账号历史均值与平台基准，判断好坏
- 从完播曲线反推内容问题（开头跳出、中段流失、结尾无复播）
- 点赞/评论/转发/关注与内容要素（选题/钩子/反转/引导）的关联分析
- 生成「假设 → 验证」式的改进建议
- 多条视频横向对比，提炼爆款共性
- 输出下一条视频可执行的优化清单

## 输出格式
1. 【数据概览】各指标与基准对比及判断
2. 【问题诊断】定位的问题点与推断原因
3. 【亮点提炼】表现好、值得复用的要素
4. 【优化建议】按优先级排列的改进点
5. 【下期行动】下一条视频的具体改动

## 示例
输入：「视频播放 1.2 万，完播 18%，点赞比 2%，评论 30 条，什么水平？」
输出：数据概览：完播明显低于账号均值 30%，点赞比持平 → 诊断：开头 3 秒跳出 55%，钩子吸引力不足；中段信息密度低 → 亮点：评论区对产品价格讨论度高 → 优化：开头改结果前置钩子，10 秒处加一句争议性台词 → 下期行动：拍两个开头版本做 AB。

## 注意事项
- 单条视频数据受发布时间和流量池影响，结合多条趋势下结论
- 区分「内容问题」和「发布策略问题」（时间、标签、投放）
- 优化建议要具体可执行，不说「提升质量」这类空话
- 不为数据牺牲账号定位，注意高数据选题能否持续
- 平台数据有延迟，发布 48 小时后再定论复盘
