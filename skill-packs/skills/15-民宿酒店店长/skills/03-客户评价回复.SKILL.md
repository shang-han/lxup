# 客户评价回复

## 适用场景
民宿/酒店店长在 OTA 平台回复住客评价时使用。涵盖好评感谢、中评说明、差评道歉补救、恶意评价应对等场景。

## 核心能力
- 好评回复：个性化感谢 + 卖点强化，引导复住与二次传播
- 中评回复：承认不足 + 说明改进 + 邀请再体验
- 差评回复：先道歉后说明，给补偿方案，展示改进行动
- 恶意/不实评价：冷静澄清事实、保留证据、申请平台处理
- 按评价内容（卫生/服务/设施/位置/噪音）调整话术侧重
- 控制回复篇幅与真诚度，避免模板化、官腔化

## 输出格式
1. 【评价分类】好/中/差/恶意及核心诉求
2. 【回复要点】针对本条的回复策略
3. 【回复文案】可直接发布的回复文本
4. 【跟进动作】内部改进或补偿行动
5. 【预防建议】避免同类问题复发的建议

## 示例
输入：「客人给差评说房间打扫不干净，床单有污渍」
输出：分类差评·核心是卫生 → 策略（真诚道歉不甩锅、说明立即排查、给出补偿）→ 文案「非常抱歉给您带来糟糕体验！卫生是我们绝不该出的疏漏，已立即对保洁重新培训并加倍检查床品更换流程。愿为您退还本次清洁费并赠送一晚免费体验券，请给我们一个改正的机会……」→ 跟进（私下联系客人补偿、排查保洁记录）→ 预防（建立逐间清洁清单交接签字）。

## 注意事项
- 24 小时内回复，差评越早回应越主动
- 不与客人争辩或人身攻击，回复是给潜在新客看的
- 补偿承诺须可兑现，不开空头支票
- 不实恶意评价用事实冷静澄清并留证向平台申诉
- 避免所有评价用同一模板，体现个性化与诚意
