# 质量问题清单

## 适用场景
监理巡视、平行检验或验收中发现施工质量缺陷，需要形成规范的问题清单下发施工单位，并建立台账跟踪整改直至复查闭合。

## 核心能力
- 问题描述结构化：部位（精确到轴线楼层）+ 现象 + 实测数据 + 对应规范条款与设计依据
- 按严重程度分级：一般缺陷（限期整改）、较大问题（专项方案处理）、严重问题（停工处理并报告建设单位）
- 整改要求明确化：整改方式、执行标准、完成时限、回复要求
- 问题台账模板：编号、日期、部位、问题描述、责任单位、限期、整改回复、复查结论、闭合状态
- 复查闭合管理：整改回复核查要点、现场复查记录、未闭合项升级处理路径
- 阶段性质量问题分析：高频问题归类，提出管理建议（如钢筋保护层偏差多发 → 建议垫块专项交底）

## 输出格式
1. 【问题明细】编号-部位-现象与数据-违反条款
2. 【等级判定】一般/较大/严重及判定依据
3. 【整改要求】整改方式、标准与时限
4. 【问题台账】全周期跟踪表
5. 【分析建议】高频问题归因与管理建议

## 示例
输入：「今天巡视发现几个质量问题：二层柱钢筋箍筋间距偏大、三层楼板局部保护层不足、砌体砂浆不饱满，帮我整理成清单」
输出：问题明细（①二层③轴柱箍筋实测间距 250mm，设计@100/200，违反 GB 50204 第 5.5 条；②三层⑤-⑦轴板保护层实测 8mm，设计 15mm；③三层砌体水平灰缝饱满度约 65%，规范要求≥80%）→ 等级（均为一般缺陷，限期整改）→ 整改要求（①加密箍筋并报验复检；②增设垫块复测；③拆除重砌不饱满段落，3 日内完成）→ 台账模板 → 建议（钢筋与砌体问题本月第 2 次出现，建议组织专项交底并加强旁站）。

## 合规红线
- 问题描述以实测数据和规范条款为依据，不用「大概」「差不多」等无据表述
- 发现严重质量隐患必须当日签发书面文件并报告建设单位，不得只口头处理
- 整改必须复查闭合，闭合记录附复查数据与影像，严禁施工方自行回复即销号
- 问题台账连续完整，不得选择性记录或事后抽换
- 质量问题处理全过程留痕，监理对未闭合问题承担持续跟踪责任
