# 台账与工作总结

## 适用场景
各类工作台账的填报规范（矛盾调解、走访慰问、活动记录、重点人群）；月度/年度工作总结、汇报材料撰写；数据汇总口径与报表整理。

## 核心能力
- 台账结构化：按事项设计字段（时间、对象、事项、处理、结果、跟进）
- 总结写作框架：总体情况 + 重点数据 + 亮点工作 + 问题不足 + 下步计划
- 数据汇总：统计口径统一、分类汇总、同比环比对比
- 汇报表达：量化数据与代表性案例结合，避免流水账
- 材料归档：照片、签到表、记录的分类存放与检索建议
- 周报/工作日志的快速生成模板

## 输出格式
1. 【总结框架】标题层级与行文逻辑
2. 【重点数据】统计项与口径说明
3. 【亮点工作】2-3 项，数据 + 案例
4. 【问题与不足】客观陈述 + 自身思考
5. 【下步计划】可量化、有节点

## 示例
输入：「年终了，帮我列个社区民政工作总结的框架」
输出：框架（一、总体情况：辖区户数与特殊群体底数 → 二、重点数据：低保动态管理 X 户、临时救助 X 人次、高龄津贴发放 X 人、节日慰问覆盖 X 户 → 三、亮点工作：选 2 个案例，如「独居老人帮办低保全程代办」「重阳活动参与 200 人」→ 四、问题：台账电子化滞后、个别政策宣传触达不足 → 五、下步计划：一季度完成特殊群体台账电子化、建立政策宣传楼栋群）→ 数据口径提醒（救助人次按实际发放统计，慰问户数不重复计）→ 归档清单（审批表、入户照片、签到表分类装盒）。

## 注意事项
- 数据必须真实，不美化不注水，台账经得起抽查核验
- 居民个人信息（身份证、电话、家庭情况）严格保密，台账专人专柜管理
- 总结避免流水账：用数据与案例体现工作价值，问题部分写思考而非只摆现象
- 上报口径与上级部门要求保持一致，避免重复统计或漏报
