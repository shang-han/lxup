# 数据分析报告

## 适用场景
运营做周期性经营复盘：周报、月报、活动专项复盘、向老板/客户汇报。把平台后台数据变成结论与可执行的改进动作。

## 核心能力
- 内容表现分析：按阅读/互动/收藏/涨粉贡献排序，识别爆款特征与低产原因
- 涨粉分析：涨粉/掉粉趋势、粉丝来源（内容/主页/搜索/直播间）、粉丝画像变化
- 互动与转化漏斗：曝光 → 点击 → 完播 → 互动 → 关注 → 转化，逐环节定位瓶颈
- 平台指标基准对照：抖音完播率/互动率、小红书收藏率/搜索占比、公众号打开率/完读率
- 竞品对标：同量级同赛道账号的更新频率、选题方向与爆款差异
- 从数据推导改进动作，生成不同颗粒度版本（周报速览/月度详报）

## 输出格式
1. 【数据概览】粉丝、发布量、总曝光/互动及环比
2. 【内容分析】Top3/Bottom3 内容及归因
3. 【粉丝分析】涨粉趋势、来源与画像
4. 【漏斗拆解】各环节转化瓶颈
5. 【竞品参考】值得借鉴的竞品动作
6. 【改进动作】下期内容/发布/互动具体调整

## 示例
输入：「小红书家居号 8000 粉，上月发 12 篇笔记，涨粉停滞，写月度报告框架」
输出：概览：12 篇（环比持平）、总阅读 24 万（环比 -15%）、涨粉 320（上月 900）→ 内容：收藏率 Top 是「清单合集」类（8.2%），纯好物图文垫底（1.1%）→ 粉丝：来源 70% 推荐页、18% 搜索——搜索占比下滑说明标题关键词弱化 → 漏斗：点击率正常但收藏/关注转化低，内容实用性不足 → 竞品：同量级账号「前后对比 + 清单」组合爆款率高 → 动作：下月清单/教程类占比提至 50%，标题前置搜索关键词（如「小卧室收纳」），笔记结尾加「收藏备用」引导，测试 2 期视频笔记。

## 注意事项
- 数据以平台后台真实统计为准，不编造不篡改汇报数据
- 注意样本量：小基数账号单条内容数据波动大，避免过度解读
- 平台指标基准随算法更新变化，跨期对比前先校准基准线
- 相关性不等于因果：涨粉停滞可能多因素叠加，逐一验证再定主因
- 报告结论须落到可执行动作，避免「加强内容质量」类空话
