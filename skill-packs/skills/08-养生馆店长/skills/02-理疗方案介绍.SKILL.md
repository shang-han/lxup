# 理疗方案介绍

## 适用场景
客户进店咨询肩颈腰背不适、睡眠差、湿气重、宫寒等调理需求时，为其设计个性化调理方案并清晰介绍，促成疗程卡成交。

## 核心能力
- 需求问询清单：主诉不适、持续时间、作息饮食、既往病史、调理目标，先问全再设计
- 方案组合设计：主项目 + 辅助项目搭配（如肩颈推拿 + 背部艾灸 + 足疗），说明搭配逻辑
- 疗程规划：总次数、建议频率、阶段目标（前段密集调理、中段巩固、后段维护）
- 方案介绍话术：用客户能理解的语言讲清「为什么这样配、多久能感觉到变化、每次做什么」
- 价格呈现技巧：单次价与疗程价对比、折算单次成本、分期/体验入口
- 方案调整机制：根据每次调理反应动态调整项目与频率的记录模板

## 输出格式
1. 【客户情况梳理】主诉与问询要点
2. 【调理思路】问题方向 + 方案逻辑（通俗表达）
3. 【方案明细】项目组合 / 次数 / 频率 / 阶段划分表
4. 【介绍话术】可直接对客使用的讲解稿
5. 【价格呈现】单次 vs 疗程对比
6. 【调整与回访节点】

## 示例
输入：「上班族肩颈僵硬、睡眠不好，想做个调理方案」
输出：问询（伏案时长、是否手麻、入睡困难还是易醒）→ 思路：肩颈经络淤堵影响头颈供血，先疏通再温养 → 方案：肩颈推拿 ×8 + 背部督脉艾灸 ×8 + 助眠足疗 ×4，前 4 次每周 2 次，之后每周 1 次 → 话术：「您这个不是一天僵的，前两周咱们先把淤堵推开，可能做完会酸一两天，是正常反应……」→ 疗程价对比单次省 30% → 第 4 次做阶段回顾再定后续。

## 注意事项
- 方案介绍不得宣称治疗疾病，禁用「治好颈椎病、治疗失眠」等表述，统一用「调理、舒缓、改善」
- 客户主诉疑似疾病症状（如持续手麻、剧烈头痛、不明原因消瘦）应先建议就医，不擅自出方案
- 疗程预期如实表达（「多数客户反馈……」「因人而异」），不做「包有效、无效退款式」疗效承诺
- 方案以客户实际需求为出发点，不堆砌高价项目
