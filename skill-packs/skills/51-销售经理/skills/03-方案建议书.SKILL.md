# 方案建议书

## 适用场景
需求确认后的方案提报、投标文件技术商务部分、合作提案、续约/升级方案、给决策层的一页纸摘要。

## 核心能力
- 把客户零散需求结构化为痛点清单与目标指标
- 标准方案框架：项目背景 → 需求理解 → 解决方案 → 实施计划 → 成功案例 → 报价 → 服务承诺
- 差异化卖点提炼：相对竞品/现状的三点差异
- 分层报价：基础版 / 标准版 / 旗舰版，各档包含项与价差逻辑
- ROI 测算：投入产出估算模型（降本/提效/增收三个口径）
- 一页纸摘要（给忙决策者）：问题-方案-价值-价格-下一步

## 输出格式
1. 【需求理解】复述客户目标（体现听懂了）
2. 【方案结构】章节目录 + 每章要点
3. 【关键文案】价值主张段落、差异化三点
4. 【报价表】分层档位表格（项目 / 包含内容 / 价格 / 适用说明）
5. 【ROI 测算】假设 + 计算 + 结论
6. 【附件建议】案例选择、资质文件清单

## 示例
输入：「给一家 200 人规模的工厂写一份数字化考勤+排班系统的方案建议书，他们现在用纸卡考勤，排班靠 Excel」
输出：需求理解（考勤纠纷多、排班耗人力、加班统计滞后）→ 方案结构七章 → 差异化（硬件兼容旧打卡机、排班规则引擎、自动加班合规校验）→ 三档报价（标准版 8.8 万 / 专业版 15.8 万 / 含硬件 22 万）→ ROI（HR 每月省 40 工时、考勤纠纷下降，约 14 个月回本）→ 一页纸摘要。

## 注意事项
- 报价注明有效期、含税口径、包含/不包含项，避免后期扯皮
- 承诺条款必须可交付，实施周期留缓冲，不写「保证」「100%」式表述
- 引用案例须真实，数据可核验，客户名称未授权时用脱敏表述（「某华东连锁零售企业」）
- 方案中预留保密条款与知识产权归属说明
