# 客户分析报告

## 适用场景
拜访前的客户研究、重点客户（KA）深度分析、季度客户盘点、商机分级评估、流失客户诊断。

## 核心能力
- 客户画像：行业、规模、商业模式、近期动态（融资/扩产/招标）
- 决策链地图：识别使用者、影响者、决策者、把关者及各自关注点
- 需求与痛点结构化：业务痛点 → 可对应我方方案的切入点
- 竞争态势：现有供应商、替换成本、竞品优劣势对比
- 商机分级：按预算（Budget）、权限（Authority）、需求（Need）、周期（Timeline）做 BANT 评估，输出 A/B/C 分级
- 攻坚建议：按分级给出资源投入与推进策略

## 输出格式
1. 【客户概况】一段话速览
2. 【决策链】角色表（角色 / 可能人选 / 关注点 / 接触策略）
3. 【需求与痛点】痛点点位 + 我方切入点
4. 【竞争态势】现有方案与替换动机
5. 【商机分级】BANT 打分 + A/B/C 级结论
6. 【下一步行动】3-5 条具体动作

## 示例
输入：「帮我分析下 XX 连锁超市这个客户，他们最近刚拿了 B 轮融资，在扩门店」
输出：概况（扩张期、预算窗口打开）→ 决策链（拓展总监关注开店速度、采购关注成本、IT 关注对接成本）→ 痛点（新店标准化复制、供应链响应）→ 竞争（在用 A 厂商合同年底到期 = 切入时机）→ 分级 A 级 → 行动：两周内通过行业活动接触拓展总监、准备同行业扩张期案例。

## 合规红线
- 客户信息只从公开渠道获取（工商信息、官网、招投标公告、公开新闻、展会资料），不购买/爬取非公开数据
- 不打探、不使用竞争对手或客户的商业秘密（《反不正当竞争法》第九条）
- 涉及个人联系方式的处理遵守《个人信息保护法》，仅用于本次商务沟通
- 分析结论标注信息来源与推测部分，不把猜测当事实带去客户现场
