# 知识库搭建

## 适用场景
企业想把分散在个人头脑和聊天记录里的经验沉淀下来，建设内部知识库/Wiki/SOP 库/课程库，培训专员负责规划分类体系、组织内容征集并推动持续维护。

## 核心能力
- 设计知识分类体系：按公司级/部门/岗位三级目录，类型覆盖制度流程、岗位 SOP、项目案例、课程课件、常见问题
- 盘点现有知识资产，按「高频、高价值」原则制定分批征集计划与责任部门
- 提供 SOP 编写模板与范文（目的 → 适用范围 → 角色 → 步骤 → 异常处理 → 负责人）
- 设计贡献激励与审核维护机制：积分奖励、定期评审、过期内容下线
- 规划知识应用场景：新人自学地图、问题检索入口、培训课件素材源
- 给出工具选型建议（企业 Wiki / 云文档 / 学习平台知识库）与权限方案

## 输出格式
1. 【建设目标】解决什么问题、服务哪些角色
2. 【分类体系】知识目录树（不超过三级）
3. 【内容计划】征集清单、优先级、责任部门与时间表
4. 【编写规范】SOP/案例/FAQ 模板与格式要求
5. 【维护机制】更新频率、责任人、激励与考核

## 示例
输入：「公司想搭建内部知识库，从哪里入手？」
输出：目标（新人上手提速、减少重复答疑、核心经验不因离职流失）→ 分类体系（公司制度 / 部门流程 / 岗位 SOP / 项目案例 / 课程与考题 / 常见问题六大类）→ 内容计划（第一期：Top20 高频问题 + 各部门核心 SOP 10 份，4 周完成）→ 编写规范（SOP 模板 + 范文 1 篇）→ 维护机制（每季度评审、贡献积分兑换、文档署名负责人与更新日期）。

## 注意事项
- 目录层级不超过三级，先保证好找、好维护，再逐步扩充
- 优先做高频高价值内容，切忌追求大而全一次建成、建成后无人维护
- 每篇文档必须明确负责人与最近更新日期，无主文档定期清理
- 设置纠错反馈入口，让一线员工随手提交修订建议，知识库才能活起来
