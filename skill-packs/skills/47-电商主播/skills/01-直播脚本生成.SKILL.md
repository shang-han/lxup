# 直播脚本生成

## 适用场景
主播或运营需要在开播前准备一份可直接照播的完整脚本，明确每个时段讲什么品、怎么过渡、何时互动抽奖，保证直播节奏流畅、不冷场。

## 核心能力
- 全流程脚本：开场暖场 → 福利引流 → 主推讲解 → 互动穿插 → 逼单 → 收尾预告
- 排品策略：引流款 / 利润款 / 爆款 / 清仓款的出场顺序与节奏
- 时间轴设计：按分钟标注每个环节时长
- 过款衔接话术：品与品之间自然过渡不突兀
- 互动节点嵌入：点赞目标、抽奖、问答时机
- 突发应对提示：冷场、黑粉、断货的临场话术

## 输出格式
1. 【直播概览】主题 / 时长 / 目标 GMV / 核心爆品
2. 【排品表】顺序 + 品类角色 + 预计讲解时长
3. 【分时段脚本】逐环节的主播口播词
4. 【互动节点】抽奖/点赞/问答的时间安排
5. 【过款衔接】品与品之间的过渡话术
6. 【收尾预告】下次直播与关注引导

## 示例
输入：「我晚上 8 点拨 3 小时，主推一款防晒霜，帮我排个脚本」
输出：概览「夏日防晒专场，3 小时，主推防晒霜」→ 排品（8:00 福利纸巾引流 → 8:15 主推防晒深度讲解 → 8:45 关联晒后修复 → 循环过款）→ 分时段口播词（开场「家人们晚上好，今天这场全是夏天刚需…」）→ 互动（每 30 分钟抽 1 单免单、点赞破 5 万解锁加赠）→ 过款衔接 → 收尾预告「明晚 8 点还有面膜专场，点个关注不迷路」。

## 合规红线
- 口播脚本禁用「最」「第一」「全网最低」等《广告法》极限词，不夸大功效
- 不虚假宣传：销量、库存、优惠、抽奖必须真实，承诺必须兑现
- 食品、保健品脚本不得宣称治疗或预防疾病的疗效
- 商业推广须标注利益相关（如广告、合作），不伪装成中立推荐
- 排品节奏张弛有度，脚本是框架而非念稿，预留临场发挥空间
