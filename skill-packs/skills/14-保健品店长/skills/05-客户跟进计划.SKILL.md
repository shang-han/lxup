# 客户跟进计划

## 适用场景
成交后的客户维护：服用指导、效果回访、复购提醒、会员关怀、沉默客户激活、售后异议处理。

## 核心能力
- 按客户分型排计划：新客（首购 7 天黄金期）、复购客（按产品耗尽周期）、会员（生日/节气关怀）、沉默客（30/60/90 天分层）
- 节点话术：服用第 3 天用法确认、第 2 周体验回访、耗尽前 5 天复购提醒、节气关怀
- 复购周期测算：按用量推算耗尽日，自动生成提醒排期
- 沉默激活：分层策略（内容唤醒 → 专属优惠 → 电话回访），避免骚扰
- 异议处理话术：「吃了没效果」「太贵了」「家人反对」等场景应答

## 输出格式
1. 【客户画像】类型、购买产品、关键日期
2. 【跟进时间线】表格：节点 / 目的 / 渠道 / 话术
3. 【复购提醒排期】预计耗尽日 + 提前提醒日
4. 【异议处理预案】2-3 个可能异议及应答
5. 【注意事项】跟进频次上限，防打扰

## 示例
输入：「顾客上周买了益生菌（30 天装），第一次买，怎么跟进」
输出：D+3 用法确认（「每天一次温水冲服记得吗？和吃药要错开两小时哦」）→ D+14 体验回访（「这两周肠胃感觉怎么样？」）→ D+25 复购提醒（「益生菌大概还能吃一周，肠道调理建议坚持 2-3 个月，老客回购有会员价」）→ 异议预案（没效果：解释肠道调理周期，建议坚持+调整作息）。

## 合规红线
- 跟进频次设上限（同一客户每周不超过 2 次主动触达），客户明确拒绝即停止
- 回访只关心服用体验，不诱导加量、不制造健康焦虑
- 「没效果」异议不得改口承诺疗效，解释个体差异与建议周期
- 客户信息仅用于跟进服务，不转卖、不泄露
