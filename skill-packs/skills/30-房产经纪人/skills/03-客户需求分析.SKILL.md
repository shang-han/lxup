# 客户需求分析

## 适用场景
经纪人接到新客咨询或录入客源：快速结构化摸清客户真实需求与意向等级，避免盲目推房与无效带看。适用于二手房、新房渠道、租房场景。

## 核心能力
- 五维需求采集：预算（总价/首付/月供承受力）、区域（刚需板块/通勤圈/学区）、户型（面积/居室/楼层朝向）、时限（急买/观望）、动机（刚需/改善/学区/投资/租房）
- 预算可行性测算：由首付反推总价、估算月供与收入要求、计入税费中介费与装修预留
- 意向等级判断（A/B/C）：A=需求明确 + 时限 3 个月内 + 预算匹配；B=有需求在对比；C=观望无时限
- 识别真需求与伪需求：区分「想要」与「必须要」，预算错配时给出取舍建议（要地段还是要面积）
- 房源匹配策略：每客推荐 2-3 套（目标房 + 对比房 + 性价比房），形成参照
- 输出需求确认单，回传客户复核避免理解偏差

## 输出格式
1. 【客户画像】基本情况与咨询背景
2. 【需求清单】五维需求结构化列表
3. 【预算测算】总价/首付/月供/税费估算
4. 【意向判断】A/B/C 分级与依据
5. 【匹配策略】推荐房源方向与匹配逻辑
6. 【待确认问题】需进一步核实的信息与回访安排

## 示例
输入：「新客电话咨询：夫妻，预算 300 万左右，想三房，近地铁，学区非硬性，希望年内入住」
输出：画像：年轻夫妻刚需首套 → 需求：总价 300 万 ±10%，三房，地铁通勤优先，学区弱需求，年内入住（关注交房与装修周期）→ 预算：按首付 3 成需 90 万 + 税费中介费约 15-20 万，月供约 1.1 万，家庭月收入需 ≥2.2 万（核实公积金可冲抵额度）→ 意向：B 偏 A（需求具体、时限明确）→ 匹配：主推 X 板块 89-95 平次新三房两套（295 万/305 万），配一套高楼层精装 315 万探预算上限 → 待确认：公积金缴存额、夫妻上班地点、是否与父母同住、现住房处置情况。

## 合规红线
- 预算建议须基于真实政策（首付比例、限贷、利率），不承诺「包办贷款」「高评高贷」
- 不诱导客户超能力购房，不参与首付贷、经营贷置换房贷等违规操作
- 客户个人信息（收入、联系方式）严格保密
- 学区、规划类信息以官方来源为准，不以不确定信息作为匹配卖点
- 客观呈现房源优劣，不为成交隐瞒房源问题
