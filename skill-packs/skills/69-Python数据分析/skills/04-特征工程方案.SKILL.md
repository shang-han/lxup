# 特征工程方案

## 适用场景
数据建模（预测/分类/用户分群）前：评估数据质量，设计清洗规则与特征构造方案，生成 Pandas/sklearn 处理代码，提升模型输入质量。

## 核心能力
- 数据质量诊断（缺失率、重复值、分布偏态、相关性）
- 缺失值与异常值处理策略（删除/填充/截尾/单独建模）
- 类别特征编码（one-hot、target encoding、embedding 建议）
- 数值特征处理（标准化、log 变换、分箱）
- 特征构造（时间特征、聚合统计、交叉特征）与特征选择（方差、相关性、重要性）
- 数据泄漏防范与训练/测试一致性

## 输出格式
1. 【数据诊断】质量问题清单
2. 【清洗方案】逐问题的处理策略与理由
3. 【特征构造】新特征列表与业务含义
4. 【处理代码】sklearn Pipeline 或 Pandas 代码
5. 【防泄漏检查】风险点与处理方式
6. 【迭代建议】特征实验方向

## 示例
输入：「用户流失预测要做特征工程，行为表有 40 列」（附字段说明）
输出：诊断：3 列缺失率超 30%、order_amount 右偏 → 清洗：高缺失列转「是否缺失」标记+中位数填充、异常值按 99 分位截尾 → 构造：RFM 特征、近 7/30 日活跃天数、优惠券使用率 → 代码：ColumnTransformer + Pipeline → 防泄漏：不得使用流失标签后产生的行为字段 → 迭代：按特征重要性筛选，尝试时间窗特征。

## 注意事项
- 清洗策略由业务含义决定，缺失本身可能就是信号
- 编码与归一化参数只用训练集学习，再应用到测试集
- 严防数据泄漏（未来信息进入特征）
- 构造的特征要能解释业务含义
- 处理流程代码化（Pipeline），保证线上线下一致
