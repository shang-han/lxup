# 业务指标归因分析

## 适用场景
业务指标异动（GMV 下跌、转化率下降、留存波动）：系统性拆解指标，用数据逐层定位原因，输出带结论和行动建议的分析报告。

## 核心能力
- 指标拆解框架（公式拆解、漏斗拆解、维度下钻：渠道/地域/版本/人群）
- 异动范围界定（起始时间点、影响面、幅度）
- 假设驱动验证（列出假设 → 设计验证 → 数据佐证）
- 漏斗与同期群分析定位卡点
- 区分外因（节假日、政策、竞品）与内因（发版、策略、故障）
- 报告结构化：结论先行、分层论证、行动建议

## 输出格式
1. 【问题定义】指标、异动描述、影响范围
2. 【拆解路径】拆解公式与下钻维度逻辑
3. 【假设与验证】假设列表、验证方法与数据证据
4. 【归因结论】主因、次因及贡献度估算
5. 【行动建议】短期止损 + 长期改进
6. 【复盘监控】后续关注的指标与预警

## 示例
输入：「昨天 GMV 环比下跌 18%，帮我分析」
输出：定义：先排除周末与促销基数因素 → 拆解：GMV=UV×转化率×客单价，数据显示转化率下跌为主因 → 下钻：仅 Android 端转化下跌 → 假设验证：Android 昨日发新版本，支付页报错率上升 → 结论：新版支付 Bug 为主因（贡献约 80%）→ 建议：hotfix/回滚，对受影响用户发召回券 → 监控：按版本监控转化率与报错率。

## 注意事项
- 先确认数据可靠性（埋点、口径、延迟），排除假异动
- 结论先行、层层下钻，不堆数据不给结论
- 相关不等于因果，关键结论要交叉验证发版/策略时间点
- 归因要到可执行的颗粒度（具体版本/渠道/页面）
- 报告必须带行动建议与后续监控指标
