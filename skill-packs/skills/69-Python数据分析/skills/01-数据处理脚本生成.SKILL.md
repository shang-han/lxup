# 数据处理脚本生成

## 适用场景
日常取数与数据处理：根据需求和表结构描述（或样例数据），编写 SQL 查询或 Pandas 脚本，完成筛选、关联、聚合、去重、缺失值处理等，并按要求输出结果。

## 核心能力
- 根据表结构与需求生成 SQL（多表 JOIN、窗口函数、子查询、CTE）
- Pandas 处理脚本（merge/groupby/pivot/apply、时间序列处理）
- 常用清洗逻辑（去重、缺失值、类型转换、异常值过滤）
- 脚本可读性：分步骤注释、口径说明
- 结果校验建议（行数核对、主键唯一性、与已知口径对账）
- 性能提示（索引、分区裁剪、内存优化）

## 输出格式
1. 【需求理解】目标、输入表、输出字段
2. 【脚本代码】SQL 或 Pandas 代码（带注释）
3. 【关键逻辑】关键处理与假设的说明
4. 【结果校验】数据质量校验方法
5. 【性能提示】数据量大时的注意点

## 示例
输入：「写个 SQL：统计上月各城市的新增用户数和次日留存率，表 users(id,city,created_at)、events(user_id,event_time)」
输出：需求理解：按注册日期做同期群统计 → SQL：CTE 先按日聚合注册，LEFT JOIN 次日活跃，GROUP BY 城市计算留存 → 关键逻辑：留存=次日活跃用户/新增用户，event_time 按日截断 → 校验：新增总数与 BI 看板对账 → 性能：events 表大建议按 event_time 分区裁剪。

## 注意事项
- 先确认口径（去重标准、时间范围边界、时区）再写代码
- JOIN 谨防行数膨胀，关联后核对行数
- 缺失值与异常值不统一删除，按业务含义处理
- 大表全表扫描必须加分区/时间过滤
- 脚本注释说明假设与口径，便于复用与交接
