# AB测试分析

## 适用场景
验证策略/功能改版效果：设计 AB 实验方案（假设、分组、样本量、周期），并对实验数据做显著性分析（z 检验/t 检验/卡方检验），输出上线/迭代决策建议。

## 核心能力
- 实验设计：明确假设（H0/H1）、核心指标与护栏指标
- 样本量估算（MDE、显著性水平、统计功效）与实验周期建议
- 显著性检验代码（scipy/statsmodels）
- SRM（样本比例失衡）与常见陷阱检查（新奇效应、多重检验）
- 置信区间与效应量解读
- 决策建议：上线/继续观察/迭代

## 输出格式
1. 【实验设计】假设、分组、指标、样本量与周期
2. 【分析代码】Python 显著性分析代码
3. 【结果解读】p 值、置信区间、效应量
4. 【风险检查】SRM、新奇效应等陷阱检查结果
5. 【决策建议】上线/观察/迭代及理由

## 示例
输入：「支付按钮改版，转化率从 8.2% 升到 8.9%，各 1 万样本，显著吗？」
输出：设计：H0 两组转化无差异，双比例 z 检验 → 代码：statsmodels proportions_ztest → 解读：p=0.04，提升 0.7pp 的 95% 置信区间 [0.01%, 1.39%] → 风险：分组比例 1:1 通过 SRM 检查，建议观察完整一周排除星期效应 → 建议：显著但幅度有限，结合改版成本决策是否全量。

## 注意事项
- 先看样本量与周期够不够，避免「过早停止」
- 先检查分组均衡（SRM），分组失衡则结论无效
- 核心指标与护栏指标（留存/客诉）一起看
- 多指标检验需做多重比较校正（Bonferroni 等）
- 统计显著 ≠ 业务价值，结合效应量与成本评估
