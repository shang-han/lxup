# 行业基础

## 行业概况
数据分析师是数据与业务决策之间的桥梁，负责取数、指标监控、专题分析、实验验证与洞察输出。Python（Pandas/NumPy）+ SQL + 可视化工具是标准技能组合，业务理解与指标体系构建能力是核心价值。岗位正从「报表工具人」转向「业务伙伴」：不只回答「发生了什么」，更要回答「为什么」和「怎么办」。

## 专业技能体系
### 硬技能
- Python数据处理（Pandas/NumPy/SQL）
- 可视化（Matplotlib/Plotly/Tableau）
- 统计分析与AB测试
- 数据清洗与特征工程
- 业务指标拆解与归因分析
### 软技能
- 分析结果讲成业务故事
- 跨部门需求对齐

## 常用术语
- 指标口径：指标的精确含义与计算标准
- 北极星指标：代表产品核心价值的最关键单一指标
- 漏斗分析：用户路径各环节的转化与流失
- 留存率：用户在 N 日后仍活跃的比例
- AB测试：随机分组对比效果，需显著性检验
- 显著性（p值）：观察到的差异由偶然产生的概率
- 归因：定位指标变化的真实原因
- 数据埋点：在关键行为处采集数据的代码 instrumentation

## 核心工作流程
需求：与业务方对齐目标 → 明确指标口径与交付物
取数：写 SQL/脚本 → 数据校验 → 构建分析数据集
分析：指标拆解 → 假设验证 → 下钻归因 → 得出结论
呈现：可视化图表 → 分析报告 → 评审宣讲
持续：指标监控看板 → 定期报表 → 专题深挖

## 常见问题
Q：业务方需求模糊怎么办？
A：先问「这个数据要支持什么决策」，用 5W 法对齐目标、口径、交付形式与时间；必要时先给一版样例结论框架确认方向。

Q：如何保证数据准确性？
A：校验口径（去重标准、时间边界、时区）、与已有报表交叉对账、关联后核对行数与主键、重要结论请同事复核。

Q：如何把分析讲成业务故事？
A：「结论 → 证据 → 建议」结构：先给结论，再用图表和数字分层论证，最后给可执行建议；避免流水账式罗列分析过程。

Q：分析结果和直觉相反怎么办？
A：先复查数据与口径（多数反直觉是数据问题）；数据可靠则深挖分群找结构性差异（辛普森悖论），再与业务方信息交叉验证。
