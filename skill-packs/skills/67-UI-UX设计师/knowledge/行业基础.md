# 行业基础

## 行业概况
UI/UX 设计师是连接产品功能与用户体验的桥梁，负责 Web、移动端与小程序的界面设计与交互逻辑。Figma 成为行业主流协作工具，设计系统与组件化交付是标配。设计师的价值正从「画得好看」转向「用设计解决问题」：以用研和数据支撑设计决策，与产品、前端紧密协作。

## 专业技能体系
### 硬技能
- 设计工具掌握（Figma/Sketch/PS/AI）
- 交互原型制作与用户测试
- 设计系统与组件库搭建
- 响应式与多端适配
- 动效设计与微交互
### 软技能
- 产品需求理解与设计沟通
- 用户视角思维

## 常用术语
- PRD：产品需求文档，设计的输入来源
- 信息架构（IA）：产品内容的组织方式与层级结构
- 线框图：低保真原型，聚焦布局与逻辑
- 高保真：接近真实产品的最终视觉稿
- 设计Token：设计系统的最小变量单位（颜色、字号、间距等）
- 可用性：用户有效、高效、满意地完成任务的程度
- 响应式：布局随屏幕宽度自适应调整
- 设计走查：上线前设计与开发共同核对还原度

## 核心工作流程
需求阶段：PRD 评审 → 需求拆解 → 设计目标确认
设计阶段：任务流程图 → 低保真原型 → 内部评审 → 高保真视觉 → 设计说明
验证阶段：可用性测试 → 设计迭代
交付阶段：标注切图 → 设计走查 → 上线验收
持续：设计系统维护 → 数据复盘 → 体验优化

## 常见问题
Q：产品/老板说「这个颜色不好看」怎么回应？
A：把主观评价拉回目标与依据：回到设计目标和用户场景，出示方案依据（数据/用研/设计原则），并提供 A/B 备选供选择。

Q：设计还原度低怎么办？
A：设计阶段提供规范标注与组件说明，上线前设计走查，与前端建立良好协作，把高频偏差沉淀进组件库根治。

Q：如何平衡美观与可用性？
A：可用性是底线，美观是加分：先保证任务效率与反馈清晰，再追求视觉表现；冲突时以可用性为先，并用测试验证。

Q：如何推动体验优化立项？
A：用数据和用研说话：把体验问题量化为业务损失（转化/留存指标），提出改进方案与预期收益，用小步 AB 验证推动。
