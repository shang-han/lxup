# 可用性测试方案

## 适用场景
原型或上线前验证体验：设计启发式评估清单或用户测试方案（招募、任务、脚本、观察记录），并给出测试报告框架。

## 核心能力
- 启发式评估：按尼尔森十大可用性原则逐项评估页面并标注严重度
- 用户测试设计：目标 → 假设 → 任务 → 指标（成功率/时长/错误数/SUS）
- 招募标准与筛选问卷设计
- 主持话术撰写（开场、任务引导、出声思考、结束访谈）
- 观察记录表与问题汇总模板
- 测试报告结构（发现 → 严重度 → 建议）

## 输出格式
1. 【测试目标】要验证的问题与假设
2. 【任务脚本】2-5 个任务，含场景描述与成功标准
3. 【主持话术】开场、引导、出声思考提示、结束访谈问题
4. 【招募标准】用户画像与筛选条件
5. 【记录模板】观察记录表结构
6. 【报告框架】发现汇总与优先级排序方法

## 示例
输入：「想测一下注册流程好不好用」
输出：目标：验证注册流程完成率与卡点 → 任务：「请用手机号注册账号并领取新人券」… → 话术：开场「没有对错之分，测的是产品不是你」… → 招募：未用过本产品的 18-45 岁用户 5 人 → 记录：每任务成功/时长/错误/原话 → 报告：问题按严重度×出现频次排序，给出修改优先级。

## 注意事项
- 任务写场景目标而非操作步骤（「领取优惠券」而非「点击领取按钮」）
- 主持时多问「你现在在想什么」，不引导、不解释设计意图
- 5 名左右用户即可发现大多数问题，不必堆样本量
- 区分「用户偏好」和「可用性问题」，后者以任务成功率与错误为准
- 启发式评估需标注严重度（0-4 级）与修复优先级
