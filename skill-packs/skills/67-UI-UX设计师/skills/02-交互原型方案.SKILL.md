# 交互原型方案

## 适用场景
视觉设计前规划页面交互框架：用文字描述各屏布局结构、元素层级、操作逻辑与状态变化，作为在 Figma/Sketch 中搭建线框图的依据。

## 核心能力
- 逐屏描述页面布局结构（导航/内容区/操作区的元素构成与层级）
- 交互逻辑描述（操作 → 响应 → 状态变化 → 跳转）
- 页面状态全集枚举（默认/悬停/点击/加载/成功/失败/空态/无权限）
- 用户旅程梳理（阶段、行为、情绪、痛点、机会点）
- 异常与手势交互（下拉、滑动、长按、快捷键）
- 动效与微交互建议点（哪里需要转场、反馈动效）

## 输出格式
1. 【交互概览】核心交互逻辑与设计理由
2. 【页面布局】各屏元素构成与层级描述
3. 【状态清单】页面/组件的全状态及对应展示
4. 【操作流】用户操作路径与系统响应
5. 【用户旅程】阶段、痛点与设计机会
6. 【待讨论点】方案中的取舍与开放问题

## 示例
输入：「设计收货地址编辑页的交互」
输出：交互概览：单页表单+底部固定保存 → 布局：顶部导航（返回/标题/保存）、表单区（姓名/手机号/地区选择器/详细地址/标签/默认开关）→ 状态清单：手机号格式错误行内提示、地区级联加载中、保存成功 Toast → 操作流：进入自动聚焦姓名、地区点击弹级联、必填未填时保存按钮置灰 → 待讨论：是否做粘贴文本智能识别地址。

## 注意事项
- 低保真聚焦逻辑而非视觉，不纠结颜色字体
- 状态全集先枚举，原型必须覆盖异常与空态
- 每个操作都要有反馈，超过 1 秒的响应需加载态
- 交互文案（按钮、提示语）随原型一并写出
- 复杂流程先画任务流程图再画页面原型
