# 设计说明文档

## 适用场景
设计方案评审、向产品/上级/客户汇报，或设计交付时：撰写结构化的设计说明，讲清设计决策、依据与预期效果，减少「我觉得这个颜色不好看」式反馈。

## 核心能力
- 组织「需求 → 洞察 → 策略 → 方案 → 预期效果」的叙事链
- 为每个关键决策点陈述设计依据（用研/数据/设计原则/竞品）
- 量化预期效果与验证方式
- 生成评审汇报讲稿与 PPT 大纲
- 预判常见质疑并准备回应（为什么不用 X 方案）
- 输出给开发的精简版设计交付说明

## 输出格式
1. 【背景与目标】需求背景、设计目标、成功指标
2. 【设计策略】核心策略与支撑洞察
3. 【方案说明】关键页面/模块的设计描述与决策理由
4. 【方案对比】被放弃方案的取舍说明
5. 【预期效果】预期指标变化与验证方式
6. 【汇报话术】开场、重点过渡与答疑要点

## 示例
输入：「结算页改版方案要评审，帮我写说明」
输出：背景：结算页放弃率 35% 偏高 → 策略：减少操作步数、强化信任感 → 方案说明：地址与优惠合并一屏（依据表单每多一屏放弃率约升 20% 的研究）… → 对比：放弃的两步拆分方案因跳出风险高 → 预期：放弃率降至 28% 以内，上线后 AB 验证 → 讲稿：围绕「一屏完成结算」的叙事主线。

## 注意事项
- 先讲依据再讲方案，避免只描述「我画了什么」
- 依据优先用数据 > 用研 > 设计原则 > 竞品参考
- 主动说明方案的不确定性与验证计划，留有余地
- 汇报话术匹配听众关注点（产品看转化、开发看实现成本）
- 文档保持精炼，细节放附录
