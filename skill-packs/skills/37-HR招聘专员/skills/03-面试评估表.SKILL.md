# 面试评估表

## 适用场景
HR/招聘专员设计面试评估工具、面试官面试前准备提问清单、面试后输出评估结论时使用。把「感觉这人还行」变成有维度、有证据、可对比的结构化评估，降低用人失误。

## 核心能力
- 评估维度设计：通用素质（沟通、学习能力、稳定性）+ 岗位胜任力（按 JD 提炼 4-6 项核心能力）
- 评分标准定义：1-5 分各分值的行为锚定描述，减少打分主观性
- STAR 提问清单：按维度设计行为面试问题（情境-任务-行动-结果），追问要点
- 面试记录模板：候选人回答的关键事实记录区，支撑评分有据可查
- 综合评估输出：优势、风险点、录用建议（强烈推荐/推荐/待定/不推荐）
- 横向对比：多候选人同维度对比表，辅助终选决策

## 输出格式
1. 【评估表头】岗位、候选人、面试官、日期
2. 【评估维度与权重】各维度评分标准（含行为锚定）
3. 【STAR 提问清单】每维度 2-3 个问题与追问要点
4. 【记录区】候选人关键回答的事实摘录
5. 【综合结论】优势 / 风险 / 录用建议与理由

## 示例
输入：「给 Java 开发工程师岗位设计一份面试评估表」
输出：维度与权重：技术功底 40%（Java 基础、框架、并发）+ 工程能力 25%（系统设计、代码质量）+ 解决问题 20%（排查与优化案例）+ 协作沟通 15% → 评分锚定：5 分「能独立设计复杂系统并讲清取舍」…… 1 分「基础概念不清」→ STAR 提问：「讲一个你优化线上性能问题的案例，当时的瓶颈是什么？你怎么定位的？最终提升了多少？」追问「如果重做会怎么改进？」→ 记录区 + 结论模板（优势/风险/录用建议）。

## 注意事项
- 【合规红线】评估只围绕岗位胜任力，不将性别、婚育、年龄、地域等作为评价维度或录用依据，违反《就业促进法》
- 面试中不得询问婚育计划、家庭背景等隐私，面试记录不记载此类信息（《个人信息保护法》）
- 评分基于候选人陈述的事实证据，避免「第一印象」「像我」等主观偏见
- 多轮面试官使用统一评估表，评估结论书面留档，便于复盘与争议追溯
- 评估表属于公司机密与候选人个人信息，注意存储权限与保密
