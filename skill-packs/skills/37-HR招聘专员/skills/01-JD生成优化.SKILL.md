# JD生成优化

## 适用场景
HR/招聘专员发布招聘岗位、优化招聘效果不佳的 JD、批量撰写多岗位说明书时使用。既要写清楚岗位要什么人，又要让 JD 对目标候选人有吸引力，同时杜绝就业歧视类违规表述。

## 核心能力
- 岗位分析：与用人部门对齐，提炼岗位核心职责（5-8 条，按重要性排序）与硬性/软性要求
- 任职要求分层：必备条件（学历、年限、技能证书）与加分项（优先条件）分开表述
- 吸引力文案：团队介绍、成长空间、福利亮点的结构化呈现，避免「高薪诚聘」式空话
- 关键词优化：嵌入招聘平台搜索关键词，提升 JD 曝光与匹配度
- 薪酬表述策略：薪资区间合理化建议、福利项的差异化呈现
- 合规自查：逐条检查是否含就业歧视表述，输出修改建议

## 输出格式
1. 【岗位名称】规范命名（避免内部黑话）
2. 【岗位职责】5-8 条，动词开头、结果导向
3. 【任职要求】必备条件 + 加分项，分层列示
4. 【薪酬福利】薪资区间与亮点福利
5. 【岗位亮点】团队/成长/文化吸引点
6. 【合规自查】歧视风险项检查结果与修改说明

## 示例
输入：「帮我写个新媒体运营的 JD，要求 1-3 年经验」
输出：职责（负责公众号/小红书内容策划与发布、社群运营、数据复盘等 6 条）→ 任职要求：本科及以上、1-3 年新媒体经验、熟悉主流平台规则（必备）；有爆款案例、会基础剪辑（加分）→ 薪酬福利：8-12K + 五险一金 + 弹性工作 → 亮点：扁平团队、内容决策权大、年度培训预算 → 合规自查：已规避「限男性/35 岁以下/已婚已育优先/限本地户口」等歧视性表述。

## 注意事项
- 【就业歧视红线】JD 不得限制性别、婚育状况、民族、宗教信仰、地域/户籍、年龄（岗位法定要求除外），否则违反《就业促进法》，可能被行政处罚并引发舆情
- 遵守《个人信息保护法》：招聘环节不超范围收集个人信息，不要求提供婚育计划、家庭背景等与岗位无关信息
- 乙肝携带者等健康信息不得作为录用限制（国家法律另有规定的特殊岗位除外）
- 职位描述客观真实，不夸大薪资与福利，避免入职后落差引发快速离职
- 硬性要求只保留真正必要的条件，门槛过高会错失合适候选人
