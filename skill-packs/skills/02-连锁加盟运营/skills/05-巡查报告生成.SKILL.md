# 巡查报告生成

## 适用场景
督导人员现场巡店后，需要把检查情况整理成规范的巡查报告，量化评分、记录问题、提出整改要求和期限，并跟踪闭环，作为加盟商考核依据。

## 核心能力
- 检查维度库：门头形象、环境卫生、产品标准、服务规范、食品安全、设备、台账
- 量化评分：按维度打分并汇总综合得分/等级
- 问题记录：问题描述 + 现场照片位 + 严重等级（红/黄牌）
- 整改要求：针对每个问题给出整改措施、责任人、期限
- 历史对比：与上次巡查结果对比，追踪整改闭环
- 报告分级：优秀/合格/待整改，挂钩加盟商积分

## 输出格式
1. 【巡查信息】门店、日期、督导员、巡查类型
2. 【评分汇总】各维度得分与综合评级
3. 【问题清单】问题 + 等级 + 照片位
4. 【整改要求】措施 + 责任人 + 完成期限
5. 【复查计划】复查时间与重点
6. 【考核结论】本次结果与积分影响

## 示例
输入：「巡店发现冷库温度偏高、两名员工没戴工帽、价签有两个缺失，出份报告」
输出：门店 XX、综合评级「待整改」→ 问题清单：① 冷库温度 -12℃（标准 ≤-18℃，红牌）② 员工未戴工帽 2 处（黄牌）③ 价签缺失 2 处（黄牌）→ 整改要求：24 小时内检修冷库并报修、当日重申着装规范、补齐价签 → 责任人店长、3 日内复查 → 考核：本次扣 5 分。

## 注意事项
- 涉及食品安全的问题（温控失效、过期、卫生）列为红牌，立即整改
- 问题描述客观具体，配照片佐证，避免主观措辞
- 整改期限合理可达成，复查形成闭环不留尾巴
- 报告作为考核依据须公正一致，同标准对所有门店
