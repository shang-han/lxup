# 运营数据分析

## 适用场景
连锁总部运营人员需要定期分析各加盟门店的营业额、客流、成本、会员等数据，横向对比找出问题门店，纵向追踪趋势，为督导重点和资源投放提供依据。

## 核心能力
- 单店诊断：营业额、客单价、来客数、毛利率、成本率的趋势与异常
- 多店横向对比：同商圈/同店龄门店排名，标出头部与落后门店
- 关键指标拆解：营业额 = 来客数 × 客单价，逐层定位短板
- 异常预警：连续下滑、成本率超标、会员流失等自动标红
- 归因分析：区分外部（商圈/天气）与内部（服务/产品/管理）原因
- 改善建议：针对落后项给出可落地的督导动作

## 输出格式
1. 【数据概览】核心指标汇总与同环比
2. 【门店排名】多店对比表（可分档）
3. 【异常门店】标红项与初步归因
4. 【指标拆解】短板环节定位
5. 【改善建议】分店/分问题的督导动作
6. 【跟踪计划】下期重点监控项

## 示例
输入：「华东 12 家加盟店上周数据，帮我找出需要重点督导的」
输出：12 店营业额排名 → 标红连续两周下滑的 3 家 → 拆解发现 A 店来客数降 18%（商圈修路）、B 店客单价降（套餐停售）、C 店成本率升至 42%（损耗失控）→ 建议：A 店加强线上引流对冲、B 店恢复招牌套餐、C 店专项查采购与损耗 → 下周重点回访 C 店。

## 注意事项
- 对比口径统一（同店龄、同商圈、同统计周期），避免误判
- 区分可干预与不可干预因素，建议聚焦门店能改变的部分
- 数据结论用于督导改进，而非单纯排名问责
- 各门店经营数据注意保密，仅在授权范围内共享
