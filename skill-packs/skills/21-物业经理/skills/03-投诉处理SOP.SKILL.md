# 投诉处理SOP

## 适用场景
物业客服、管家接到业主投诉、报修、邻里纠纷、对服务不满等诉求时，需要一套标准、可落地的处理流程，确保「件件有回应、事事有闭环」，降低投诉升级与舆情风险。

## 核心能力
- 投诉分级：按紧急程度/影响范围分为一般/重要/重大三级，匹配响应时限
- 标准五步流程：受理登记 → 分类派单 → 处理执行 → 结果反馈 → 回访归档
- 各类诉求的响应时限建议（如漏水等急修 30 分钟内到场、一般报修 24 小时）
- 疑难/重复投诉的升级机制（管家→主管→项目经理→公司/业委会）
- 群体性投诉与网络舆情的应急处置要点
- 生成可填写的投诉处理登记表与回访记录模板

## 输出格式
1. 【诉求识别】类型 + 分级 + 紧急程度
2. 【响应时限】该级别的规定动作与时间节点
3. 【处理流程】分步骤 SOP（谁、做什么、何时）
4. 【沟通话术】受理与反馈口径
5. 【升级条件】何时上报、报给谁
6. 【闭环表单】登记表 / 回访记录模板

## 示例
输入：「顶楼业主投诉楼上漏水，墙上都发霉了，很生气，怎么处理？」
输出：识别为「重要-房屋漏水」类 → 响应时限（2 小时内管家+工程部上门勘查）→ 流程（登记→现场勘查取证→查明漏水源（楼上住户/公共管道/外墙）→责任界定→协调维修→验收→72 小时回访）→ 受理话术（先共情安抚、承诺到场时间）→ 升级条件（责任争议大或涉及结构安全转项目经理）→ 附投诉登记表模板。

## 注意事项
- 先安抚情绪、再处理事情，第一时间给出明确到场/反馈时限
- 责任未界定前不轻易表态归责，避免引发连带赔偿争议
- 全程留痕（照片、记录、签字），作为后续依据
- 处理结果必须回访确认，未获业主认可不销单
- 涉及人身安全（燃气、电路、结构）的按应急预案优先处置
