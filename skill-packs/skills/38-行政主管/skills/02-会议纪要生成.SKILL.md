# 会议纪要生成

## 适用场景
行政主管/文员在经营例会、部门会议、专题会议、管理层会议后整理纪要时使用。把冗长的讨论记录提炼成结论清晰、任务到人、可追踪的正式纪要，让会议成果落地而不是开完就忘。

## 核心能力
- 记录转纪要：从流水账式记录中剥离闲聊与跑题，提炼核心议题与结论
- 决议归纳：区分「讨论中」「已决议」「待跟进」，明确每个议题的最终结论
- 行动项管理：每项待办落实到「任务 + 责任人 + 完成时间 + 交付标准」四要素
- 纪要分级：按会议性质调整详略（经营会重决议、专题会重过程结论）
- 敏感信息处理：涉及人事、薪酬、未公开决策的表述把握
- 规范格式：标题、编号、时间地点、参会/缺席、记录人、签发等要素齐全

## 输出格式
1. 【会议要素】名称、时间、地点、主持人、参会人、记录人
2. 【议题与决议】逐议题列出讨论要点与最终结论
3. 【行动项清单】任务 / 责任人 / 截止时间 / 交付标准表格
4. 【遗留问题】未决事项与下次跟进安排
5. 【签发信息】抄送范围与签发人

## 示例
输入：「把这段经营例会记录整理成纪要：讨论了下季度预算、市场部要增加 20 万投放、张总说先批 12 万看效果、小李下周三前出投放方案……」
输出：纪要：会议要素齐全 → 议题一「下季度预算」：决议——市场部投放预算增批 12 万元（原申请 20 万），先试行一个月按 ROI 评估是否追加 → 行动项表：① 投放方案（小李，下周三前，含渠道与预算拆分）② 首月效果复盘（市场部，次月 5 日前）→ 遗留问题：剩余 8 万待首月数据后二次评审 → 抄送：管理层、财务部。

## 注意事项
- 纪要记结论不记过程，冗长讨论只保留影响决策的关键观点
- 行动项必须四要素齐全，缺责任人或截止时间的待办等于没写
- 涉及敏感决策（人事任免、薪酬）控制抄送范围，按需脱敏
- 纪要 24 小时内发出，时效过了就失去督办价值
- 关键纪要请主持人/领导签发确认后再分发，避免结论偏差
