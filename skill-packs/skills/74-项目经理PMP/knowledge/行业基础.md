# 行业基础

## 行业概况
项目经理是在受限资源下对项目目标达成负责的角色，管理范围、进度、成本、质量、风险与干系人。传统行业（建筑、制造、工程、能源等）以项目制运作为主，PMP 知识体系（PMBOK）是通用方法论；软件与创新业务广泛采用敏捷（Scrum/Kanban），混合模式（整体里程碑规划 + 执行层迭代）已成为主流。岗位核心矛盾：在不确定性与变更中守住目标，平衡多方利益。

## 专业技能体系
### 硬技能
- WBS工作分解与甘特图排期
- 风险识别与应对策略
- 成本预算与挣值管理（EVM）
- 干系人分析与沟通计划
- 敏捷/Scrum/Kanban方法论
### 软技能
- 跨部门推动与向上汇报
- 团队冲突化解

## 常用术语
- WBS（工作分解结构）：把项目范围逐层分解为可管理工作包的结构化方法
- 关键路径：活动网络中最长的路径，决定项目总工期，延误即整体延误
- EVM（挣值管理）：用 PV/EV/AC 与 CPI/SPI 综合衡量进度与成本偏差的方法
- 基准：经批准的范围/进度/成本版本，偏差衡量的标尺
- 变更控制：对基准的任何修改须经申请-评估-审批流程并留痕
- 干系人：能影响项目或被项目影响的个人与组织
- Sprint（冲刺）：Scrum 中固定时长的迭代周期，通常 1-4 周
- Kanban（看板）：可视化工作流管理工具，核心是限制在制品（WIP）与度量流动

## 核心工作流程
启动：项目章程 → 干系人识别 → 启动会
规划：范围定义 → WBS → 进度与预算 → 风险计划 → 沟通计划
执行与监控：任务分派与跟踪 → 进度/成本监控（EVM）→ 变更控制 → 风险评审 → 定期汇报
收尾：交付物验收 → 项目复盘 → 经验教训归档 → 团队释放

## 常见问题
Q：项目已经确定延期，怎么补救？
A：先明确剩余范围与新目标日期 → 压缩关键路径（赶工/快速跟进，优先关键路径活动）→ 走变更流程削减低价值范围 → 申请资源 → 向发起人报告现状与选项，切忌瞒到最后。

Q：职能部门不配合、资源调不动怎么办？
A：先查项目章程与启动会是否明确了资源承诺（向上授权是否足够）；日常量化「不配合对项目目标的影响」借发起人协调；关系层面与资源部门找共同利益点（如联合署名成果、优先级交换）。

Q：客户临时加需求怎么处理？
A：不当场拒绝也不当场答应：记录 → 评估对范围/进度/成本的影响 → 给出取舍选项（加费用、延期、或置换等量原有范围）→ 客户书面决策 → 更新基准并通知团队。

Q：敏捷和 PMP 矛盾吗？
A：不矛盾：PMP 提供管理框架（范围/风险/干系人/沟通/采购），敏捷提供执行节奏（迭代/快速反馈/持续改进）。传统行业项目常用混合模式：总体里程碑规划 + 执行层迭代交付 + 管理层传统汇报。
