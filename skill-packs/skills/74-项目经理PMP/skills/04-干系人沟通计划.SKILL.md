# 干系人沟通计划

## 适用场景
项目启动时识别干系人并制定差异化沟通策略；项目周报/月报/向上汇报材料撰写；干系人出现抵触或分歧时的沟通应对；关键里程碑的沟通安排。

## 核心能力
- 干系人识别与分析：权力-利益方格、态度评估（支持/中立/抵触）
- 沟通计划：对象、目的、频率、方式、内容模板、责任人
- 项目周报/月报撰写：进度、风险、需决策事项、下步计划四段式
- 向上汇报结构：结论先行、一页纸报告、带选项与建议的决策项
- 抵触干系人策略：利益绑定、高频透明沟通、升级路径
- 会议管理：议程、纪要模板、决议跟踪表

## 输出格式
1. 【干系人地图】清单 + 权力-利益分类 + 当前态度
2. 【沟通策略】分类施策（重点管理/令其满意/随时告知/最少关注）
3. 【沟通矩阵】频率/方式/内容/责任人
4. 【模板产出】周报或向上汇报大纲
5. 【敏感点提醒】高风险干系人的沟通要点

## 示例
输入：「老板平时不关注项目，开会时突然质问进度，怎么改善向上沟通？」
输出：诊断（缺定期向上汇报机制，信息不对称导致不信任）→ 沟通矩阵（双周一页纸书面报告 + 每月 15 分钟当面汇报 + 红色风险即时上报）→ 一页纸模板（总体状态红绿灯 + 3 个关键进展 + 2 个风险 + 需决策事项及选项）→ 汇报话术建议（先结论后过程，带着「建议选 A，因为…」而不是罗列问题）→ 敏感点（老板关注成本，每次汇报固定带 EVM 成本快照）。

## 注意事项
- 向上汇报先给结论与选项，不做过程流水账
- 重要决策与承诺必须书面纪要确认，口头共识无据可查
- 坏消息早报：风险暴露的时机越晚，处置成本越高
- 干系人态度动态变化，重大里程碑后更新干系人地图
