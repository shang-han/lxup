# 客户需求分析

## 适用场景
定制师接到新客户咨询时，需要通过一套结构化的提问与倾听，快速摸清客户的出行目的、偏好、预算与限制条件，避免凭感觉设计导致方案反复返工。

## 核心能力
- 需求访谈提纲：出行人群 / 时间 / 预算 / 目的 / 偏好 / 禁忌六大维度
- 客户画像建模：亲子、情侣、银发、团建、蜜月等典型画像特征
- 显性需求与隐性需求区分（说想去景点，其实想放松）
- 预算敏感度与品质要求判断
- 限制条件识别：假期长度、身体状况、忌口、预算上限
- 输出需求清单，作为行程设计的输入

## 输出格式
1. 【基本信息】出行人数/关系、时间、天数
2. 【出行目的】度假/旅拍/亲子/团建等核心诉求
3. 【偏好清单】喜欢的景点类型、美食、节奏、住宿档次
4. 【限制条件】预算、体力、忌口、时间硬约束
5. 【客户画像】典型画像归类与关键特征
6. 【设计建议】给行程方案的方向性指引

## 示例
输入：「客户说想带爸妈出去玩，但没说具体，帮我列个沟通提纲」
输出：访谈提纲（① 爸妈年龄和身体状况，能否长时间走路/坐长途 ② 想国内还是出境、几天假期 ③ 预算范围 ④ 爸妈偏好山水/历史/还是休闲度假 ⑤ 有无慢病或忌口）→ 画像「银发孝心游，核心是轻松不累+陪伴」→ 限制（节奏要慢、住宿要好、避免高原和长途奔波）→ 设计建议（优选气候温和、医疗便利、景点集中的目的地，行程留足休息时间）。

## 注意事项
- 多问开放式问题，让客户多表达，别急于推产品
- 关注同行者的真实需求（带老人孩子时尤其重要）
- 预算要委婉但明确地问清区间，避免方案跑偏
- 隐性需求往往比显性需求更关键，要善于追问「为什么」
- 把需求落成文字清单与客户确认，减少理解偏差
