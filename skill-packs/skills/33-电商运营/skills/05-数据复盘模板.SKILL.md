# 数据复盘模板

## 适用场景
电商运营做周期性经营复盘（日报/周报/月报）、大促活动后复盘，或流量/转化异常下滑时的诊断。提供结构化模板与分析框架，帮助快速定位问题环节并输出改进动作，避免「只罗列数据、没有结论」。

## 核心能力
- 多周期模板：经营日报/周报/月报、大促活动复盘、单品诊断报告
- 核心指标体系：流量（UV/PV）、转化（转化率/加购率）、成交（GMV/客单价）、利润（毛利/ROI）、服务（DSR/退款率）
- 漏斗拆解：曝光 → 点击 → 进店 → 加购 → 支付，定位问题卡在哪个环节
- 归因分析框架：环比、同比、竞品对比、行业均值四向对标，逐层定位原因
- 数据 → 结论 → 行动的闭环：每个异常指标配原因假设与验证方法
- 输出可执行的改进清单：动作、负责人、截止时间、预期效果

## 输出格式
1. 【复盘周期与目标】周期、核心目标及完成率
2. 【核心指标概览】GMV / UV / 转化率 / 客单价 / ROI 的环比、同比对比
3. 【漏斗分析】异常环节定位与初步归因
4. 【亮点与问题】做得好的（可复用经验）与不足的（待改进点）
5. 【行动计划】改进措施 + 负责人 + 截止时间 + 预期效果
6. 【下周期目标】下一阶段关键指标与重点动作

## 示例
输入：「这周店铺 GMV 下滑 30%，帮我复盘」
输出：套用周报复盘模板 → 指标概览：GMV 35 万 → 24.5 万（-30%），拆解为 UV 降 25%、转化率降 7% → 漏斗定位：曝光稳定但点击率下降，怀疑主图点击疲劳或竞品降价分流 → 验证方法：查主图更换记录、对比竞品价格带变化 → 行动计划：更换主图（设计，周三前）、报名平台限时补贴活动（运营，周四前）→ 下周期目标：GMV 回升至 32 万。

## 注意事项
- 指标口径统一（如转化率是否剔除退款），避免跨周期数据不可比
- 数据要有对标（环比/同比/行业均值），孤立数字没有意义
- 区分相关与因果，先验证归因再下行动结论
- 复盘的目的是行动，每个结论必须落到可执行任务上
- 关注平台规则变更、大促节奏等外部因素对数据的干扰
