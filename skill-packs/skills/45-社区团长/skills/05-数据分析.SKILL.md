# 数据分析

## 适用场景
团长想搞清楚生意好不好、哪款卖得动、社群是否健康、佣金收入如何，需要从每日/每周/每月的经营数据中读出问题并指导选品和运营。

## 核心能力
- 核心指标解读：GMV、佣金收入、客单价、转化率、复购率、退货率
- 选品分析：各品类销量/毛利/退货率排名，识别爆品与滞销品
- 社群健康度：活跃度、下单率、退群率、新增成员趋势
- 时段分析：找出推单最佳时间与截单时间
- 问题诊断：销量下滑、转化低、复购差的归因
- 优化建议：从数据推导选品、定价、推单节奏的调整

## 输出格式
1. 【数据概览】核心指标汇总与环比变化
2. 【亮点】表现好的品类/单品/动作
3. 【问题】数据暴露的短板与归因
4. 【优化建议】选品 / 定价 / 推单 / 维护的具体调整
5. 【下期目标】可量化的改进目标

## 示例
输入：「这周卖了 3200 元，比上周少了，帮我分析下」
输出：概览（GMV 3200，环比 -18%，客单价 32 元，下单 100 单）→ 归因（本周水果到货延迟缺货，爆款脐橙断供 2 天，带动整体下滑）→ 问题（选品过度依赖单一爆品、到货时间不稳）→ 建议（拓展 2-3 个备用爆品、与供应商确认到货时效、缺货日提前预告改推替代品）→ 下期目标（GMV 回升至 3800，爆品备货充足率 100%）。

## 注意事项
- 分析要落到具体动作，避免只罗列数字不给建议
- 区分正常波动与趋势性下滑，别被单日数据误导
- 关注退货率与复购率，它们比单次 GMV 更能反映健康度
- 数据口径要一致（同一时间范围、同一统计方式）才能环比
- 把数据结论转化为下一次选品和推单的决策依据
