# 客户维护SOP

## 适用场景
团长积累了稳定的社群用户后，需要系统化的客户维护流程来提升复购、处理售后、沉淀老客，从「推货的」转变为「邻居信任的团长」。

## 核心能力
- 客户分层标准：按消费频次/客单/活跃度分为活跃、潜力、沉默、流失四类
- 分层维护动作：活跃客晒单激励、潜力客专属优惠、沉默客唤醒触达
- 复购激活节奏：下单后 3 天回访、7 天复购提醒、节庆专属关怀
- 售后处理标准：坏果包赔、质量问题退款、缺货补发的流程与话术
- 老客标签管理：记录偏好、忌口、家庭结构，做精准推荐
- 转介绍激励：老带新奖励机制设计

## 输出格式
1. 【客户分层】分层标准与对应人群
2. 【维护动作】每层的触达频率与内容
3. 【复购节奏】关键时间节点的提醒动作
4. 【售后流程】问题分类 → 处理标准 → 话术
5. 【执行清单】可勾选的每日/每周维护任务

## 示例
输入：「我群里老有人买了一次就不来了，怎么维护」
输出：先做客户分层找出「买过一次」的潜力客 → 复购节奏（首单后第 3 天私信问体验、第 7 天推关联品+老客专属券）→ 维护话术「姐，上次团的橙吃完了吗？这周到了批更新的，老客户给你留了张 2 元券～」→ 售后兜底承诺降低决策顾虑 → 每周清单：回访 10 位沉默客、晒单奖励 3 位活跃客。

## 注意事项
- 售后响应要及时（建议 2 小时内），先安抚再解决，不在群内推诿
- 赔付标准提前明确并公示，避免每次扯皮
- 私信维护比群发更有温度，重点客户一对一沟通
- 记录客户偏好要做长期台账，复购推荐才精准
- 承诺的优惠和赔付必须兑现，信任是团长的核心资产
