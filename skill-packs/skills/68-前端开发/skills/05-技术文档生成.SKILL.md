# 技术文档生成

## 适用场景
项目开发、组件封装或技术选型后需要补文档：生成结构化技术文档，让组件好用、方案好评审、代码好维护。

## 核心能力
- 组件文档：功能、props/events 表格、使用示例、常见问题
- 技术方案：背景、方案对比、详细设计、风险与排期
- 项目 README：快速开始、目录结构、技术栈、部署说明
- 代码注释补充：函数级注释与复杂逻辑说明
- 变更日志 / Release Notes 整理
- API 接口文档整理

## 输出格式
1. 【文档类型与读者】明确文档定位
2. 【正文】完整文档内容（标题、表格、代码示例）
3. 【示例代码】可运行的示例
4. 【待补充项】需作者补充的占位（具体数值/版本）
5. 【维护建议】文档更新时机与责任人

## 示例
输入：「帮这个 Upload 组件写使用文档」（附组件代码）
输出：组件简介与适用场景 → props 表（accept/maxSize/multiple/onSuccess 等含类型与默认值）→ 示例：基础上传、多文件+大小限制、自定义触发区 → FAQ：上传失败排查、进度事件说明 → 待补充：后端接口地址 → 维护：组件变更时同步更新文档。

## 注意事项
- 文档以「怎么用」开头，示例可直接运行
- props/API 表格须含类型、默认值、说明三列
- 技术方案重点写方案对比与决策依据，不只记录结论
- 注释解释「为什么」而非「是什么」
- 文档与代码保持同步，过时文档比没有文档更糟
