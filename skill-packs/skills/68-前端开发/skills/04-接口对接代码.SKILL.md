# 接口对接代码

## 适用场景
对接后端接口时：根据接口文档（Swagger/OpenAPI）生成请求封装、数据模型类型、状态管理代码，以及统一的 loading/error/空态处理。

## 核心能力
- 根据接口文档生成请求函数与 TS 类型定义
- 请求层封装（Axios/fetch 拦截器、统一错误码、token 处理）
- 状态管理设计（Redux Toolkit/Zustand/Pinia 的数据流）
- 异步状态处理（loading/success/error/empty、失败重试）
- 分页、缓存与重复请求取消的处理
- Mock 数据与本地联调建议

## 输出格式
1. 【数据模型】TS 类型定义
2. 【请求层】请求函数与拦截器关键代码
3. 【状态设计】Store 结构与 actions
4. 【组件接入】组件侧调用与状态处理示例
5. 【异常处理】统一错误码处理与用户提示策略
6. 【Mock 建议】本地 mock 方案

## 示例
输入：「对接订单列表接口，GET /api/orders 支持分页，Vue3+Pinia」
输出：类型：Order、PageResult<Order> → 请求：usePageList 封装（参数合并 + 竞态只保留最新请求）→ Pinia store：list/total/page/loading/error + fetchOrders action → 组件：onMounted 首取、翻页重载、错误态重试按钮 → 错误码：401 跳登录、5xx 统一 Toast → Mock：vite 插件 + 本地 json。

## 注意事项
- 接口字段在前端做语义化命名转换，避免耦合后端字段变动
- 异步竞态要处理（快速翻页只保留最新结果）
- 错误处理统一收口，避免每个页面散落 try/catch
- 敏感操作考虑幂等与防重复提交
- 与后端约定错误码与枚举，代码中注释版本
