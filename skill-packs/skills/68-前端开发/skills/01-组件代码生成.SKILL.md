# 组件代码生成

## 适用场景
开发新页面/组件时：根据设计稿或需求描述，按项目技术栈（React/Vue + TypeScript）生成组件代码骨架，含 props 定义、状态设计、事件处理与常见边界处理。

## 核心能力
- 按需求生成 React（Hooks + TS）或 Vue 3（Composition API + TS）组件代码
- Props/Events 接口设计（含类型定义与默认值）
- 状态管理与数据流组织（父子通信、状态提升）
- 常见处理：loading/空态/错误态、防抖节流
- 可访问性基础（语义化标签、键盘操作、aria 属性）
- 附使用示例与关键注释

## 输出格式
1. 【组件设计】职责、props/events 接口、状态清单
2. 【代码实现】完整组件代码（带 TS 类型）
3. 【使用示例】父组件调用示例
4. 【边界处理】loading/空数据/异常的处理说明
5. 【扩展建议】后续可扩展方向

## 示例
输入：「用 React+TS 写一个搜索下拉组件，支持防抖联想和键盘选择」
输出：接口设计：props { placeholder, options, loading, onSelect } → 代码：useDebounce Hook + 受控 input + ArrowUp/Down 高亮选项 + Enter 选中 → 使用示例 → 边界：无匹配空态、请求竞态只保留最新一次、失焦关闭面板 → 扩展：大数据量可接虚拟滚动。

## 注意事项
- 生成代码须符合项目现有技术栈与 lint 规则，先统一风格
- Props 命名语义化，避免暴露内部实现
- 异步请求等副作用须处理清理与竞态
- 列表渲染使用稳定 key，避免用 index 作 key
- 生成代码需人工审查关键逻辑，AI 产出仅作参考
