# 性能优化方案

## 适用场景
页面加载慢、滚动卡顿、内存占用高，或专项性能治理：基于症状和测量数据（Lighthouse/Performance 面板）定位瓶颈，按优先级输出优化方案。

## 核心能力
- 加载性能：首屏优化（代码分割、懒加载、预加载、CDN、缓存策略）
- 渲染性能：长任务、回流重绘、虚拟列表、requestAnimationFrame
- 资源优化：图片压缩与格式（WebP/AVIF）、字体加载、Tree Shaking
- 框架层面：React memo/useMemo/useCallback 的合理使用、Vue computed
- 测量与验收：Lighthouse 指标（LCP/INP/CLS）、Performance 面板分析方法
- 按投入产出比排优先级输出方案

## 输出格式
1. 【问题诊断】症状 → 可能瓶颈 → 验证方法
2. 【优化清单】逐项：优化点、预期收益、实施成本、优先级
3. 【实施步骤】关键优化的操作步骤与代码示例
4. 【验收指标】优化前后对比的指标
5. 【长期机制】性能监控与防劣化建议

## 示例
输入：「H5 商城首屏 4 秒，LCP 3.8s，怎么优化？」
输出：诊断：bundle 1.2MB 过大 + 首屏大图未压缩 → 清单：①路由级代码分割（LCP 预计-1s，成本低，P0）②首图 WebP+懒加载（P0）③接口预请求（P1）④静态资源 CDN+强缓存（P1）→ 步骤：React.lazy + Suspense 示例 → 验收：LCP < 2.5s、bundle < 500KB → 长期：接入 Lighthouse CI 防劣化。

## 注意事项
- 先测量再优化，不凭感觉猜瓶颈
- 优先做低成本高收益项（图片、缓存、代码分割）
- memo/useCallback 不是万能的，避免过早优化增加复杂度
- 优化后要用数据验证（前后指标对比）
- 移动端弱网场景必须真机实测
