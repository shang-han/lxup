# 行业基础

## 行业概况
前端开发负责构建 Web、H5、小程序与跨端应用的用户界面与交互逻辑。React 与 Vue 两大生态为主流，TypeScript 已成标配，工程化（Vite/Webpack）、组件化与性能优化是核心能力。岗位边界不断扩展：向上延伸至 BFF 与 Node 服务，向下深入可视化与跨端，AI 辅助编程普及后，工程质量与产品理解成为竞争关键。

## 专业技能体系
### 硬技能
- HTML/CSS/JavaScript核心能力
- React/Vue框架与组件开发
- 浏览器兼容与性能优化
- API对接与状态管理
- Git协作与代码规范
### 软技能
- 产品逻辑理解与边界情况思考
- 文档与注释习惯

## 常用术语
- SPA：单页应用，页面无需整体刷新即可切换
- 组件化：将界面拆分为独立可复用的组件
- 虚拟DOM：UI 的内存表示，框架 diff 更新的基础
- 首屏时间：从导航到首屏渲染完成的时间
- LCP/CLS/INP：Core Web Vitals 核心指标（加载/视觉稳定/交互响应）
- 防抖/节流：控制函数执行频率的策略
- 状态管理：共享数据的集中管理（Redux/Pinia/Zustand 等）
- Code Review：代码互审，保障质量与传递规范

## 核心工作流程
需求：需求评审 → 明确交互与边界情况 → 工时评估
开发：组件拆分 → 页面开发 → 接口联调 → 自测
协作：分支管理（Git Flow）→ 提交 → Code Review → 合并
交付：联调测试 → 修复 Bug → 走查 → 上线
持续：性能监控 → 线上问题排查 → 技术债清理

## 常见问题
Q：如何系统思考边界情况？
A：沿数据生命周期枚举：空数据、加载中、失败、超时、无权限、极值、重复操作、断网；每个接口和表单都问一遍「失败/为空/变慢会怎样」。

Q：组件复用性怎么提高？
A：单一职责、控制反转（slot/render props）、受控与非受控都支持、props 语义化；基础组件不耦合业务，重复三次以上再抽象。

Q：浏览器兼容问题怎么处理？
A：先明确目标浏览器范围并查 Caniuse，用 Babel/Autoprefixer 配置 polyfill 与 targets，重点关注 iOS 低版本 WebView 与小程序环境，真机测试。

Q：Git 协作怎么更高效？
A：小提交、写清楚提交信息；勤开分支勤合并；统一 Conventional Commits 规范；用 pre-commit 钩子跑 lint/test 拦截基础问题。
