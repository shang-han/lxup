# 脚本生成

## 适用场景
视频剪辑师/短视频创作者在拍摄前需要完整脚本：口播视频、剧情短片、产品种草、知识科普、企业宣传等。输出可直接照着拍摄或剪辑的脚本文本，解决「有想法写不成稿」的问题。

## 核心能力
- 按平台适配时长与节奏：抖音/快手（15-60 秒强钩子）、视频号（1-3 分钟）、B 站（中长视频）
- 黄金开头设计：前 3 秒钩子类型库（悬念、冲突、利益点、反常识、痛点提问）
- 内容结构模板：痛点 → 原因 → 方案 → 证明 → 行动号召；SCQA、起承转合等结构
- 口播文案口语化：短句、断句、可直接朗读，标注语气与停顿
- 商业脚本植入：产品介绍自然融入剧情/知识，避免硬广感
- 完播与互动设计：在脚本中预埋互动点与结尾引导（关注/评论/收藏）

## 输出格式
1. 【选题定位】目标人群、核心信息、视频时长、平台
2. 【开头钩子】前 3 秒台词 + 画面建议
3. 【脚本正文】分段台词，标注【画面】【字幕】【语气】
4. 【结尾引导】行动号召与互动设计
5. 【拍摄备注】场景、道具、机位等执行提示

## 示例
输入：「写个 30 秒口播脚本，卖挂脖风扇，投抖音」
输出：钩子「40 度高温还在外面跑的打工人，这条视频能救你的命」（前 3 秒怼脸 + 擦汗画面）→ 正文：痛点（挤地铁汗流浃背）→ 产品出场（挂脖即用、解放双手）→ 卖点三连（三档风 / 一充一天 / 不绞头发）→ 信任背书（已售 10 万+）→ 结尾「点下方小黄车，今天拍一发二」→ 备注：快节奏剪辑，每个卖点配特写 + 花字。

## 注意事项
- 前 3 秒决定划走率，钩子必须与内容相关，不做标题党
- 口播文案写完要朗读测试，拗口的长句一律拆短
- 一条视频只讲一个核心信息，信息过载等于没讲
- 商业内容遵守《广告法》，禁用绝对化用语，功效宣称需有依据
- 脚本预留剪辑节奏空间，标注画面切换点便于后期
