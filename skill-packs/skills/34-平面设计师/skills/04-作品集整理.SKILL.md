# 作品集整理

## 适用场景
平面设计师求职、晋升答辩、接商单建立信任或年度作品沉淀时整理作品集。解决「作品很多但不会讲故事」「只会放图没有思考」的问题，让作品集体现专业能力而非只是图库。

## 核心能力
- 项目筛选策略：按目标（求职/接单）选 6-10 个代表作，宁缺毋滥，覆盖能力面
- 结构化叙事框架：项目背景 → 设计挑战 → 设计策略 → 方案呈现 → 落地效果
- 项目文案撰写：把每个项目写成「解决了一个什么问题」的故事，而非图片堆叠
- 视觉呈现建议：展示图顺序（主视觉 → 细节 → 应用场景 → 实物样机）、留白与节奏
- 数据化表达：用可量化结果背书（转化率提升、销量增长、获奖、客户复购）
- 分类编排：按项目类型（品牌/包装/海报/电商）或行业分组，附个人简介与技能栈

## 输出格式
1. 【整理策略】作品集目标、目标受众、筛选原则
2. 【项目清单】入选项目与排列顺序、每个项目一句话定位
3. 【项目文案模板】背景 / 挑战 / 策略 / 方案 / 效果五段式（逐项目生成）
4. 【展示建议】每项目建议展示的图量与图序
5. 【整体结构】封面 → 个人简介 → 项目区 → 联系方式的成册结构

## 示例
输入：「我要找品牌设计师的工作，手头有 15 个项目，怎么整理作品集」
输出：策略：求职作品集选 7 个最能体现品牌设计能力的项目，弱化纯美工类 → 清单与排序（以完整品牌案开场压阵）→ 示例文案：「XX 茶饮品牌重塑——背景：老品牌形象老化，年轻客群流失；挑战：保留品牌资产的同时年轻化；策略：提炼『东方轻养生』概念，重塑 LOGO 与色彩体系；方案：主视觉 + 30 项应用规范；效果：门店焕新后季度营收提升 22%」→ 建议每项目 6-8 张图（主视觉 1 + 细节 2 + 应用 3 + 样机 2）→ 成册结构建议。

## 注意事项
- 质量优先于数量：放 10 个平庸项目不如 6 个精品，弱项目果断舍弃
- 每个项目都要有「为什么这样做」的思考，只放图等于自证是美工
- 使用真实数据背书，无法量化时用客户评价、获奖、落地照片替代
- 涉密项目（未上市产品、签有 NDA 的客户）需脱敏或取得授权再放
- 作品集格式匹配投递渠道：求职用 PDF（控制 20MB 内）、接单用在线链接
