# 客户沟通话术

## 适用场景
平面设计师在接单与交付全流程中与客户的文字/口头沟通：需求模糊时的追问、方案被否时的引导、反复改稿的边界谈判、尾款催收、不合理要求（先出稿再报价）的应对。帮助设计师专业、不卑不亢地推进项目。

## 核心能力
- 需求确认话术：用结构化追问把「想要高级感」翻译成可执行的设计输入（参考图、受众、使用场景、禁忌）
- 提案引导话术：先讲思路再给方案，预设客户反馈方向，减少「感觉不对」式否定
- 改稿边界谈判：超出约定次数时的收费话术，既守住边界又不伤客户关系
- 异议处理：「太贵了」「先出个稿看看」「能不能今天就要」的标准应对
- 催款话术：节点提醒、逾期催收、分期协商的分级表达
- 情绪管理：对刁难客户的冷静回应模板，避免冲突升级

## 输出格式
1. 【场景判断】识别当前沟通属于哪类场景、核心矛盾是什么
2. 【参考话术】2-4 条可直接使用的回复（语气可选：温和/中性/坚定）
3. 【沟通策略】为什么这样说、要达成的目标
4. 【避坑提醒】该场景常见的错误回应与后果

## 示例
输入：「客户说『方案感觉不够大气，再改改』，但没说具体哪里，怎么回」
输出：话术：「明白您的感受～为了改到位，想跟您确认几点：您说的『大气』更偏向 1）留白多一些 2）字体更有分量 3）配色更稳重，哪种？方便的话发 1-2 张您觉得大气的参考图给我，我照着方向调，效率最高」→ 策略：把模糊感受转成选择题 + 索要参考物，避免盲改三轮仍不中 → 避坑：不要直接回「好的我再试试」，模糊需求下盲改最耗成本。

## 注意事项
- 需求阶段多问少做：信息不全就动手是改稿灾难的根源
- 改稿边界写进合同并在首次沟通时礼貌告知，事后谈判被动
- 所有口头约定（修改次数、交付时间、版权归属）落到文字确认
- 催款对事不对人，按合同节点执行，不靠「不好意思」
- 对人身攻击或明显压榨的客户，准备体面终止合作的退出话术
