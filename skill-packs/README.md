# 岗位技能包（skill-packs）

龙虾优盘「岗位技能包」制作源目录。**51 个岗位 × 每岗 5 个技能 = 255 个技能包**
技能包三引擎（Hermes / OpenClaw / Codex）共享，不区分引擎——选岗位后三个框架都加载同一套技能。

## 目录结构

```
skill-packs/
├── manifest.json          技能包索引（版本 + 内容指纹 sha256）
├── 51-posts.json          岗位总表（51 个）
└── skills/
    ├── 01-餐饮店长/
    │   ├── post.json          岗位元数据（名称/行业/技能清单）
    │   ├── skills/            技能文件（SKILL.md 格式，每岗 5 个）
    │   ├── knowledge/         行业知识库
    │   └── scripts/           可执行脚本/插件（预留）
    ├── 02-连锁加盟运营/
    ├── ...
    ├── 50-心理咨询师/
    └── 51-销售经理/
```

## SKILL.md 格式

YAML frontmatter + Markdown 正文，纯文本零依赖：

```markdown
---
name: 药品知识问答
description: "一句话说明技能做什么"
version: 1.0.0
triggers: [触发词1, 触发词2]
platform: all
---

# 技能名
## 适用场景 / ## 核心能力 / ## 输出格式 / ## 示例 / ## 注意事项（强监管行业为「合规红线」）
```

技能文件会被引擎作为上下文注入，`triggers` 用于技能匹配/广场搜索。
医药、保险、房产、教培、医美养生、直播带货、心理咨询等强监管岗位的技能内置**合规红线**。

## 使用流程（与前端/启动器对接）

1. 技能页展示 `51-posts.json` / `manifest.json` 中的岗位包名称、图标、技能列表
2. 用户「下载安装」→ 拉取岗位目录（正式版 AES 加密传输，本地解密）→ 校验 sha256
3. 安装即写入引擎技能目录（`engines/<engine>/skills/`），三引擎共享，选岗位后加载对应技能包
4. `locked: true` 岗位需授权解锁后方可使用

## 校验指纹

岗位目录全部文件按路径排序拼接后取 sha256，修改任一文件后需重算并更新 manifest.json：

```bash
cd skills && find 13-药店店长 -type f | sort | xargs cat | sha256sum
```
